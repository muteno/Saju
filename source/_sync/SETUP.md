# 팀즈 대화 자동 모으기 — 설정 설명서

이 문서대로 하면, 내 PC가 일정 시간마다 팀즈 대화를 가져와 깃에 자동으로 올린다.
한 번만 설정해두면 그다음은 알아서 돌아간다.

> **누가 하면 되나?** PC를 좀 다뤄야 하는 단계(아래 1~2단계)가 있어서, 개발을 맡은 사람이
> 진행하길 권한다. 설정이 끝나면 누구도 손댈 필요 없이 자동으로 돈다.

> **용어 미리 풀기**
> - **테넌트(tenant)** = 우리 회사/팀의 마이크로소프트 공간 전체를 가리키는 이름.
> - **앱 등록(app registration)** = "이 프로그램이 우리 대화를 읽어가도 된다"고 마이크로소프트에 미리 신고해 두는 절차. 신고하면 열쇠(ID) 두 개를 준다.
> - **권한(permission)** = 그 열쇠로 "무엇까지" 할 수 있는지. 우리는 "내 대화 읽기"만 준다.

---

## 1단계 — 마이크로소프트에 프로그램 신고하기(앱 등록)

1. 브라우저에서 **https://entra.microsoft.com** 접속 → (관리자 계정으로 로그인).
2. 왼쪽 메뉴에서 **앱 등록(App registrations)** 클릭 → 위쪽 **+ 새 등록(New registration)** 클릭.
3. **이름** 칸에 아무 이름이나: 예) `yeul-chat-sync`.
4. **지원되는 계정 유형**은 맨 위 **"이 조직 디렉터리의 계정만"** 선택.
5. 맨 아래 **등록(Register)** 클릭.
6. 등록되면 나오는 화면(개요/Overview)에서 두 값을 복사해 둔다:
   - **애플리케이션(클라이언트) ID** → 나중에 `clientId` 칸에 넣음
   - **디렉터리(테넌트) ID** → 나중에 `tenantId` 칸에 넣음

## 2단계 — "내가 직접 로그인" 방식 켜기

1. 방금 만든 앱 화면에서 왼쪽 **인증(Authentication)** 클릭.
2. 아래로 내려 **고급 설정(Advanced settings)** → **퍼블릭 클라이언트 흐름 허용(Allow public client flows)** 을 **예(Yes)** 로 바꾼다.
3. 위쪽 **저장(Save)** 클릭.

## 3단계 — "내 대화 읽기" 권한 주기

1. 왼쪽 **API 권한(API permissions)** 클릭 → **+ 권한 추가(Add a permission)**.
2. **Microsoft Graph** 클릭 → **위임된 권한(Delegated permissions)** 클릭.
3. 검색창에 `Chat.Read` 입력 → 체크 → **권한 추가(Add permissions)**.
   - (로그인할 때 본인이 동의하므로 별도 관리자 동의는 보통 필요 없다. 조직 정책상 필요하면 같은 화면의 "관리자 동의 부여"를 눌러도 된다.)

## 4단계 — 설정 파일 만들기

1. 이 폴더(`source/_sync/`)의 **`config.example.json`** 을 복사해, 같은 폴더에 **`config.json`** 이라는 이름으로 저장한다.
2. `config.json` 을 열어 1단계에서 복사한 값을 채운다:
   - `tenantId` ← 디렉터리(테넌트) ID
   - `clientId` ← 애플리케이션(클라이언트) ID
   - `chatId` ← **일단 비워두자.** (다음 단계에서 찾는다)

## 5단계 — 어떤 대화를 모을지 고르기

1. 윈도우 시작 메뉴에서 **PowerShell** 을 연다.
2. 이 폴더로 이동: `cd` 뒤에 이 폴더 경로를 붙여 엔터. 예)
   `cd C:\Users\내이름\Yeul\source\_sync`
3. 아래를 치고 엔터:
   `./sync-teams-chat.ps1`
4. 처음이라 로그인 안내가 뜬다 — 화면에 나온 **주소로 가서 코드를 입력**하면 끝.
5. 로그인 후, `chatId` 가 비어 있으면 **내 대화 목록과 각 대화의 ID** 를 쭉 보여준다.
   모으고 싶은 대화의 ID 를 복사해 `config.json` 의 `chatId` 에 넣고 저장한다.

## 6단계 — 처음으로 진짜 모아보기

다시 `./sync-teams-chat.ps1` 실행.
- `../chats/` 폴더에 대화가 마크다운으로 저장되고, 깃에 올라가면 성공.
- (깃에 올리려면, 이 PC에 이 저장소가 내려받아져 있고 push 가 되는 상태여야 한다.)

## 7단계 — 자동으로 돌게 예약하기(작업 스케줄러)

> "작업 스케줄러" = 윈도우가 정해진 시간마다 프로그램을 대신 눌러주는 알람 같은 기능.

1. 시작 메뉴에 **작업 스케줄러(Task Scheduler)** 검색 → 실행.
2. 오른쪽 **기본 작업 만들기(Create Basic Task)** 클릭.
3. 이름 `yeul-chat-sync` → 다음.
4. **트리거(언제 돌릴지)**: "매일" 또는 "컴퓨터 시작 시" 등 원하는 주기 선택 → 다음.
   (자주 쓰는 시간대에 1~2시간마다가 무난하다.)
5. **동작**: "프로그램 시작" 선택 → 다음.
6. **프로그램/스크립트** 칸: `powershell.exe`
   **인수 추가** 칸: `-ExecutionPolicy Bypass -File "C:\...\source\_sync\sync-teams-chat.ps1"`
   (따옴표 안은 이 스크립트의 실제 전체 경로로 바꾼다.)
7. 마침. 이제 정해진 때마다 알아서 대화를 모아 깃에 올린다.

## 솔직한 주의사항 몇 가지

- **이 스크립트는 처음 한 번은 직접 돌려보며 확인이 필요하다.** 환경마다 응답 형식이
  조금씩 달라서, 실제 테넌트에서 한 번 테스트한 뒤 자동 예약을 걸자.
- **개인 대화를 모으는 일이다.** 시작 전에 참여자들에게 동의를 한 번 받자.
- **비밀값은 깃에 안 올라간다.** `config.json`·`token-cache.json`·`sync-state.json` 은
  `.gitignore` 로 막아뒀다 — 이 파일들은 절대 공유하지 말 것.
- **대화 본문은 쉐어포인트/원드라이브로는 안 나온다.** 본문은 위 방식(Graph)로만 가져올 수 있다.
  (원드라이브엔 대화에 '첨부한 파일'만 들어간다.)
- 막히는 데가 있으면 그 화면을 캡처해서 물어보면, 다음 단계까지 같이 짚어줄게.
