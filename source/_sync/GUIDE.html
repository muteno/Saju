팀즈 대화 → 깃 자동 수집 · 설치 가이드

한 번만 설정해두면, PC가 정해진 시간마다 팀즈 대화를 알아서 가져와 깃(코드 창고)에 쌓아준다.

첫 1회만 한 번 해두면 끝(다시 안 함) 자동 컴퓨터가 알아서 이미 있으면 스킵 이미 돼 있으면 건너뛰기
큰 그림 한 줄. 아래 A(첫 1회 설정)를 한 번 끝내면, 그다음부턴 B(자동)로 굴러간다. 깃 설치·앱 등록·관리자 동의·스케줄 등록은 전부 “한 번만” 하는 일이다.
순서 🖱️ 쉬운 길 — .bat 더블클릭 추천 A0. 시작 전 — 참여자 동의 첫 1회 A1. 깃 설치 + 저장소 내려받기 첫 1회 있으면 스킵 A2. 마이크로소프트에 앱 등록 (열쇠 2개 받기) 첫 1회 A3. “내가 직접 로그인” 방식 켜기 첫 1회 A4. “내 대화 읽기” 권한 + 관리자 동의 첫 1회 A5. 설정 파일(config.json) 만들기 첫 1회 A6. 어떤 대화 모을지 고르기 첫 1회 A7. 한 번 진짜로 모아보기(테스트) 첫 1회 A8. 자동으로 돌게 예약 첫 1회 B. 그다음부터 — 자동 자동 곁가지: 딱 한 번만 통째로 받고 싶다면 (Graph Explorer) 막혔을 때 / 솔직한 주의
🖱️ 더 쉽게 — 명령어 타이핑이 싫으면 “.bat 더블클릭”
설정(A1~A5)을 한 번 끝낸 뒤에는, source\_sync 폴더의 파일을 더블클릭만 하면 된다:
  • run-sync.bat — 더블클릭 = 대화 한 번 수집 (A6·A7을 대신)
  • install-schedule.bat — 더블클릭 = “2시간마다 자동” 등록 (A8을 대신)
  • uninstall-schedule.bat — 더블클릭 = 자동 실행 해제
대화를 실제로 긁어오는 두뇌는 PowerShell(sync-teams-chat.ps1)이라, .bat은 그 파일과 같은 폴더에 있어야 한다. 첫 실행 때만 로그인 코드 입력 한 번 필요(A6).

A0시작 전 — 참여자 동의 첫 1회

여기 쌓이는 건 참여자들의 실제 대화다. 모으기 전에 한 번만 물어보고 가자:

“우리 대화를 작품 재료로 모아둘게, 괜찮을까?” → 다들 OK면 진행.

A1깃 설치 + 저장소 내려받기 첫 1회 이미 깃 쓰면 스킵

깃(Git) = 코드를 깃허브(인터넷 창고)에 올리고 내리는 도구. 이미 PC에서 깃을 쓰고 클론·푸시가 된다면 이 단계 전체를 건너뛰어도 된다.

① 깃 설치 (윈도우)

  1. 브라우저에서 열기 → https://git-scm.com/download/win
  2. 다운로드된 설치 파일 실행 → 설정 바꿀 것 없이 Next 계속 → Install → Finish

② 잘 깔렸는지 확인

  1. 시작 메뉴에서 PowerShell 검색 → 열기
  2. 입력 후 엔터: 
    git --version
  3. git version 2.x … 처럼 나오면 성공. (안 나오면 PowerShell 창을 닫고 새로 열기)

③ 내 이름표 달기 (한 번만)

git config --global user.name "네이름"
git config --global user.email "muteno@pm.me"

④ 저장소 내려받기

# 문서 폴더로 이동
cd $HOME\Documents
# 저장소 통째로 받기
git clone https://github.com/exitmaru/Yeul.git
받는 도중 깃허브 로그인 창이 브라우저 팝업으로 뜬다 → 내 깃허브 계정으로 로그인/승인. 이게 “올리기(push) 권한”을 받는 과정이라 첫 1회만 한다.
끝나면 Documents\Yeul 폴더에 저장소가 내려와 있다. 안에 source\_sync 폴더가 보이면 성공.

A2마이크로소프트에 앱 등록 — 열쇠 2개 받기 첫 1회

앱 등록 = “이 프로그램이 내 대화를 읽어가도 된다”고 마이크로소프트에 신고하는 절차. 신고하면 열쇠(ID) 두 개를 준다.

  1. 열기 → https://entra.microsoft.com (관리자 계정으로 로그인)
  2. 왼쪽 메뉴 앱 등록(App registrations) → 위쪽 + 새 등록(New registration)
  3. 이름: yeul-chat-sync (아무거나)
  4. 지원되는 계정 유형: 맨 위 “이 조직 디렉터리의 계정만” 선택
  5. 맨 아래 등록(Register)
  6. 등록 후 뜨는 개요(Overview) 화면에서 두 값을 복사해 둔다:
    • 애플리케이션(클라이언트) ID → 나중에 clientId
    • 디렉터리(테넌트) ID → 나중에 tenantId

A3“내가 직접 로그인” 방식 켜기 첫 1회

  1. 방금 만든 앱 화면 → 왼쪽 인증(Authentication)
  2. 아래로 내려 고급 설정(Advanced settings) → 퍼블릭 클라이언트 흐름 허용(Allow public client flows) → 예(Yes)
  3. 위쪽 저장(Save)

A4“내 대화 읽기” 권한 + 관리자 동의 첫 1회

  1. 왼쪽 API 권한(API permissions) → + 권한 추가(Add a permission)
  2. Microsoft Graph → 위임된 권한(Delegated permissions)
  3. 검색창에 Chat.Read 입력 → 체크 → 권한 추가(Add permissions)
  4. 관리자 동의(필요 시): 같은 화면의 “<조직>에 대한 관리자 동의 부여” 클릭 → 예. 첫 1회 한 번 눌러두면 끝.
로그인할 때 본인이 동의하므로 관리자 동의가 꼭 필요하진 않다. 조직 정책상 막혀 있으면 위 4번을 눌러 한 번만 풀어주면 된다.

A5설정 파일(config.json) 만들기 첫 1회

  1. 탐색기로 Documents\Yeul\source\_sync 폴더 열기
  2. config.example.json 복사 → 같은 폴더에 config.json 으로 저장
  3. config.json 열어서 채우기: 
    {
  "tenantId": "A2에서 복사한 디렉터리(테넌트) ID",
  "clientId": "A2에서 복사한 애플리케이션(클라이언트) ID",
  "chatId":   "일단 비워두기 (A6에서 채움)",
  "branch": "main",
  "autoCommit": true,
  "autoPush": true
}
config.json · token-cache.json · sync-state.json 은 비밀값이라 깃에 안 올라간다(.gitignore로 막아둠). 공유 금지.

A6어떤 대화 모을지 고르기 첫 1회

  1. 시작 메뉴 → PowerShell 열기
  2. 폴더로 이동: 
    cd $HOME\Documents\Yeul\source\_sync
  3. 스크립트 실행: 
    powershell -ExecutionPolicy Bypass -File .\sync-teams-chat.ps1
    (-ExecutionPolicy Bypass = “이 스크립트 한 번 실행 허용”. 윈도우 기본 보안 때문에 붙임.)
  4. 처음이라 로그인 안내가 뜬다 → 화면에 나온 주소로 가서 코드 입력(딱 한 번).
  5. 로그인 후 chatId가 비어 있으면 내 대화 목록 + 각 대화의 ID를 보여준다. 모으고 싶은 대화의 ID를 복사해 config.json의 chatId에 넣고 저장.

A7한 번 진짜로 모아보기 — 테스트 첫 1회

다시 같은 명령으로 실행:

powershell -ExecutionPolicy Bypass -File .\sync-teams-chat.ps1
  • source\chats\ 폴더에 대화가 마크다운으로 저장되고, 깃에 올라가면 성공.
  • 깃허브 저장소의 source/chats/에 파일이 보이면 끝.

🖱️ 명령어가 번거로우면 폴더의 run-sync.bat을 더블클릭해도 똑같이 실행된다.

여기서 한 번은 꼭 직접 돌려봐야 한다. 이 스크립트는 실제 테넌트에 연결해 검증된 적이 없는 “출발점”이다. 에러가 뜨면 그 메시지를 그대로 복사해 알려주면 바로 고쳐줄 수 있다.

A8자동으로 돌게 예약 — 작업 스케줄러 첫 1회

🖱️ 가장 쉬운 방법: 폴더의 install-schedule.bat을 더블클릭하면 “2시간마다 자동”이 한 번에 등록된다. (아래 수동 방법은 대안)

작업 스케줄러 = 윈도우가 정해진 시간마다 프로그램을 대신 눌러주는 알람.

  1. 시작 메뉴에 작업 스케줄러(Task Scheduler) 검색 → 실행
  2. 오른쪽 기본 작업 만들기(Create Basic Task)
  3. 이름 yeul-chat-sync → 다음
  4. 트리거(언제): “매일” 또는 “컴퓨터 시작 시” 등 → 다음 (자주 쓰는 시간대에 1~2시간마다가 무난)
  5. 동작: “프로그램 시작” → 다음
  6. 프로그램/스크립트: powershell.exe
    인수 추가: -ExecutionPolicy Bypass -File "C:\Users\내이름\Documents\Yeul\source\_sync\sync-teams-chat.ps1" (따옴표 안은 실제 전체 경로로 교체)
  7. 마침

B그다음부터 — 자동 자동

이제 손댈 게 없다. 예약한 시간마다 PC가 알아서:

  • 저장된 토큰으로 자동 로그인(다시 코드 입력 안 함)
  • 지난번 이후 새 메시지만 가져와 source\chats\에 쌓고
  • 깃에 자동으로 올린다
즉 A를 한 번 끝내면, B는 영원히 자동이다. 가끔 로그인이 만료되면 그때만 다시 코드 한 번 입력하면 된다.

곁가지 — 딱 한 번만 통째로 받고 싶다면 (Graph Explorer)

스크립트 세팅 없이, 지금 당장 대화를 한 번 긁어오는 방법. 붙여넣기보다 깔끔하다.

  1. 열기 → https://developer.microsoft.com/graph/graph-explorer
  2. 오른쪽 위에서 회사 계정으로 로그인
  3. 입력 후 Run: 
    GET https://graph.microsoft.com/v1.0/me/chats
    → 내 대화 목록이 뜸 (권한 동의 창 뜨면 Chat.Read 허용)
  4. 원하는 대화의 id 복사
  5. 입력 후 Run: 
    GET https://graph.microsoft.com/v1.0/chats/{복사한id}/messages?$top=50
    → 대화가 JSON으로 쫙. 복사하면 끝.
  6. 대화가 길면 결과 맨 아래 @odata.nextLink 주소를 다시 Run → 다음 묶음

막혔을 때 / 솔직한 주의

  • 스크립트 테스트(A7)는 직접 한 번 돌려봐야 한다. 환경마다 응답이 조금씩 달라서, 실제로 한 번 확인한 뒤 자동 예약을 걸자.
  • 개인 대화를 모으는 일이다 — 시작 전 참여자 동의(A0).
  • 비밀값은 공유·커밋 금지: config.json, token-cache.json, sync-state.json.
  • 대화 본문은 쉐어포인트/원드라이브로는 안 나온다. 본문은 위 방식(Graph)으로만. 원드라이브엔 대화에 ‘첨부한 파일’만 들어간다.
  • 어느 단계든 막히면 그 화면을 캡처하거나 에러 메시지를 그대로 복사해서 물어보면, 다음 단계까지 같이 짚어준다.
