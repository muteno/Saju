# 명리학 도사 — 사주풀이 대화 웹앱 (미연시풍) 설계

> 확정: 2026-07-16. 이 문서가 이 앱의 단일 설계 기준. "절대 원칙"의 변경은 사용자 승인 필요.

## 무엇을 만드나

내담자(화자)가 생년월일시를 말하면, 이 저장소에 아카이빙된 사주명리 지식(정제 코퍼스 2,504글)을
근거로 '명리학 도사' 캐릭터가 비주얼노벨(미연시)풍 대화로 풀이해주는 웹앱.
배포: Cloudflare Pages(이 저장소에 이미 연결돼 있음 — 앱이 생기면 빌드 실패도 해소) + Pages Functions.

## 절대 원칙 (사용자 확정)

1. **근거는 무조건 아카이브 자료.** 풀이의 모든 주장에는 코퍼스 출처가 있어야 한다.
2. **자료를 찾는 주체는 LLM이 아니라 사주 데이터.** 만세력 계산 결과(일주·십신·신살·합충·대운…)가
   지식 베이스의 조회 키가 된다. 임베딩·유사도·자유 검색(RAG) 금지 — **결정론적 키 조회만**.
   (사주는 수학처럼 완전 정립된 학문이 아니어서 검색 유사도로는 정확한 풀이가 불가능하다는 판단.)
3. **LLM 없이도 해석은 완결 동작.** 조립식 리포트가 기반층. LLM은 그 위의 서술·대화층 —
   앱이 골라 건네준 근거 유닛을 받아 그때그때 도사 화법으로 풀어낸다. LLM은 서술만, 자료 선택 안 함.
4. **지식은 원문 더미가 아니라 요소별 정제 유닛으로 고도화.** 증류 시 문헌 간 이견은 뭉개지 말고
   `관점차이`로 병기 → 도사가 "이 문헌은 이리, 저 문헌은 저리 본다"를 종합.

## 레이어 구조

```
L1 만세력 엔진 (TS·클라이언트)  생년월일시·성별·양음력·출생지 → 원국 4주·지장간·십신·
                                십이운성·신살·합충형파해·대운(방향/대운수)·세운·월운 = "키셋"
L2 지식 베이스 (정적 JSON)      키 → 정제 유닛. 2,504글을 오프라인 증류(출처 병기)
L3 해석 조립기 (TS)             키셋 → 유닛 조립 → 섹션 리포트 (LLM 불필요·단독 완결)
L4 도사 대화층 (Pages Function → Claude API)  페르소나+차트+리포트+근거 유닛 주입
L5 미연시 UI                    대화창·도사 캐릭터·선택지·만세력 표 (forceteller-ref 참고)
```

### L1 만세력 엔진 — 정밀도 요건 (정확도의 심장)

- 연주·월주 경계는 **절입 시각** 기준 (입춘이 연주 경계)
- 절기표 1900–2100: Meeus 태양황경 계산으로 오프라인 생성(파이썬) → 정적 JSON(KST 시각)
- 시주: 야자시/조자시 옵션 (23시 경계의 일주 귀속 선택)
- 진태양시 보정: 출생지 경도(서울 ≈ -32분) + **한국 서머타임 이력(1948–60, 1987–88) 반영**
- 대운: 성별 × 연간 음양 → 순/역행, 절입까지 일수÷3 = 대운수
- 검증: 포스텔러(pro.forceteller.com) 결과와 표본 대조 (`forceteller-ref/README.md`의 API 구조 참고)

### L2 지식 베이스 — 키 체계 (1차)

| 키 공간 | 예 | 수량 | 주 소스(코퍼스 장) |
|---|---|---|---|
| `ilju/*` | ilju/갑자 | 60 | 일주론(안녕사주명리·초코서당) |
| `cheongan/*` `jiji/*` | cheongan/갑 | 10+12 | 천간과 지지(2개 소스) |
| `sipsin/*` | sipsin/편관 | 10 | 십신과 합충, 사주공부-명리 |
| `sibiunseong/*` | sibiunseong/제왕 | 12 | 십이운성 |
| `sinsal/*` | sinsal/역마 | ~30 | 신살(3개 소스) |
| `hapchung/*` | hapchung/자오충 | ~40 | 십신과 합충 |
| `yongsin/*` | 강약·조후 | 소수 | 원국 분석 및 용신 |
| `unse/*` | unse/2026병오, 월운/일간별 | 연·월 | 연도별 운세, 이달의 운세 |
| `image/*` | image/sipsin/정관 | 46 | **역추론의 순방향화** — "이 요소면 겉으로 이렇게 보일 확률" (실전 리딩 영상에서 확인된 층. 인상 스캔 2,969문단) |
| `frame/*` `chain/*` | frame/신강, chain/식상생재 | 상호작용 색인 103키 | 구조 판정·십신 체인·합충 조합 (바둑 수읽기 층) |

유닛 스키마(안):

```json
{ "key": "ilju/계유", "title": "계유일주",
  "sources": [{"doc": "일주론 글모음", "unit_id": "…", "excerpt": "…"}],
  "distilled": { "핵심": "…", "성격": "…", "직업": "…", "관계": "…", "주의": "…",
                 "관점차이": ["A 문헌은 …, B 문헌은 …"] } }
```

생성 파이프라인: `refine-tools/units.json` + 라벨 → 제목·라벨 기반 키 매핑(자동) → Claude 배치 증류 → 검수.
글 제목이 요소명 그대로인 경우가 많아(일주론 60편 등) 매핑은 대부분 기계적.

### L3 해석 조립기

섹션 순서: 원국 표 → 일주 → 십신 구성(강약) → 합충 → 신살 → 십이운성 → 대운 흐름 → 올해 세운.
문장마다 출처 표기 유지. **이 층까지가 LLM 없는 완결 MVP.**

### L4 도사 대화층

**서술 표준 (사용자 확정, 2026-07-16)** — "일반인 언어로 풀고, 근거를 달아라":
1. 일상어 풀이가 본문 — 전문용어는 비유·생활어로 번역 (예: 일지 = '나의 안방')
2. 생활 장면으로 번역 — 언제, 어떤 상황에서, 어떻게 나타나는지
3. 시기 구체화 — '유년'이 아니라 '2029년 기유년' (엔진이 환산)
4. 완충 요인과 대처까지 — "그래서 어떻게 하면 되는지"에 반드시 답할 것
5. 단락 끝에 ▸근거줄: 전문용어 + 출처(보드/문헌/엔진) 병기
6. 단정 금지 — 경향 표현("~하기 쉽습니다", "패턴이 반복될 수 있습니다")

- Pages Function이 Anthropic API 프록시 (키는 서버측 환경변수, 클라이언트 노출 금지)
- 프롬프트 = 도사 페르소나 + L1 차트 JSON + L3 리포트 + 이번 질문에 해당하는 근거 유닛 원문
- 후속 질문: 인텐트 → **고정 주제 분류표** 안에서 키 선택(예: 직업운 → 관성·재성·대운) →
  L2 결정론 조회 → 근거 주입. 자유 검색 금지 원칙은 여기서도 동일.
- LLM 미설정/호출 실패 시: L3 리포트를 대화체로 분할 출력 (완결 동작 원칙)

### L5 미연시 UI

비주얼노벨 대화창(타이프라이터 효과), 도사 스탠딩(대기/골몰/풀이 표정), 주제 선택지 버튼,
만세력 표는 포스텔러 레이아웃 참고(`forceteller-ref/`).

## 구현 단계

1. **L1 만세력 엔진** — 절기표 생성기 → 간지·십신·신살·대운 계산 → 포스텔러 표본 대조 ← **다음 작업**
2. **L2 증류 1차** — 키 매핑 자동 추출 + 일주 60 · 십신 10 · 천간지지 22부터
3. **L3 조립 리포트 + 웹 배포** (LLM 없는 MVP 완성 시점)
4. **L4 도사 대화** (Claude API)
5. **L5 미연시 연출 고도화** (캐릭터 일러스트·연출)

## 미정 (추후 사용자와 결정)

- 도사 캐릭터 컨셉(연령·성별·말투)과 일러스트 확보 방식
- 격국 자동 판정 포함 여부 (문헌 이견 큼 — 1차는 강약·조후 중심 보수 판정)
