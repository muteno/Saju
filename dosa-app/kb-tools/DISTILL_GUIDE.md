# 증류 지침서 (배치 작업자용)

목적: 코퍼스 원문(키별 번들)을 요소별 정제 유닛(JSON)으로 증류한다.
이 유닛이 도사의 근거이자, LLM 없는 조립 리포트의 본문이 된다.

## 절대 규칙 (위반 시 검증기가 잡는다)

1. **원문에 없는 내용을 쓰지 않는다.** 모든 항목은 번들 원문에서 나와야 한다. 일반 상식·모델 지식으로 보충 금지.
2. **`인용`은 원문 축자(1~2문장 그대로).** 한 글자라도 다르면 검증 실패. 출처 유닛 키를 정확히.
3. **연도별 운세 단락은 제외** (2020년~2026년 운세 등 — unse/* 유닛이 따로 다룸). `제외메모`에 남긴다.
4. **문헌 간 이견은 뭉개지 말고 `관점차이`에 병기.** 이견이 없으면 서술 각도 차이라도 1건 남긴다(구조분석 vs 물상 등).
5. 블로그 안내문(링크 참조 안내, 포스팅 예고, 검색 유입 인사)은 지식이 아니므로 버린다.
6. 문체: 중립·간결(명사형/평서). 도사 말투 아님 — 말투는 L4가 입힌다.

## 작업 절차 (키 1개당)

1. `kb/distill_src/<키공간>_<이름>.md` 번들을 읽는다 (## [출처 유닛키] 단위로 구분됨).
2. `kb/distilled/<키공간>/<이름>.json` 을 쓴다 — **`kb/distilled/ilju/갑자.json`이 표준 예시**. 스키마:
   - `key`, `title`, `elements`(ilju만: 일간/일지/일지십신/십이운성/공망 — 원문 명시값 사용, 엔진과 대조됨)
   - `sources`: 번들의 모든 출처 나열 `{unit, doc, title}`
   - `distilled`: `핵심`(2~3문장) / `성격`[] / `직업`[] / `관계`[] / `주의`[] / `물상`(1~2문장; 원문에 물상 서술이 있을 때)
   - `인용`: 1~3건 축자 `{text, src}`
   - `관점차이`: `[{주제, 견해: [{src, 내용}]}]`
   - `기타`: 유명인·참고문헌 등 (있을 때만), `제외메모`
3. 검증: `python3 dosa-app/kb-tools/validate_distilled.py` — **자기 파일 관련 오류만** 수정 (다른 파일 오류는 무시).

## 품질 기준

- 성격/직업/관계/주의 각 3~7개 항목. 근거 밀도 높게(원문 표현 유지 우선, 압축은 하되 의역 왜곡 금지).
- 괄호로 근거 구조 병기 권장: "(일지 정인 — 명예·안정 지향)" 식.
- 원문이 부실한 키(짧은 글 1건뿐 등)는 있는 만큼만 쓰고 `기타`에 "원문 소략" 명시.

## 이미지(image/*) 유닛 — 역추론의 순방향화

목적: "이 요소가 발달한 사람은 **겉으로 이렇게 보일 확률이 있다**" — 도사가 생일만 듣고
"주변에서 이런 소리 듣지 않나?"를 선제 발화하게 하는 지식층. 번들은 `ix_image_*.md`.

스키마 (elements 없음):
```json
{ "key": "image/sipsin-group/인성", "title": "인성이 발달한 사람의 대외 이미지",
  "sources": [ {"unit": "yt:g_pf1FHQGVs", "doc": "유튜브", "title": "…"},
               {"unit": "일주론 글모음#0001", "doc": "…", "title": "…"},
               {"unit": "board", "doc": "방법론 보드", "title": "궁성"} ],
  "distilled": {
    "핵심": "…(2문장)",
    "겉모습": ["외모·분위기·스타일 경향", …],
    "말투와 태도": ["대화·행동에서 드러나는 것", …],
    "오해와 실제": ["겉과 속이 다른 지점 — '차가워 보이지만 실제로는…'", …],
    "활용": ["이 이미지를 살리는 법(자기PR·직업 연결)", …] },
  "인용": [{"text": "축자", "src": "yt:vid 또는 유닛키 또는 board"}],
  "관점차이": [ … ] }
```

추가 규칙:
1. **순방향 문장만** — "~로 보일 확률이 있습니다/보이는 경향" (역추론 서술 금지)
2. 성격 일반론 반복 금지 — 겉으로 **드러나는** 것만 (기존 sipsin/cheongan 유닛과 중복 회피)
3. 유튜브 인용은 `yt:<vid>` 출처로 자막 원문 축자 (STT 오탈자도 그대로)
4. 보드 인용은 `board` 출처
