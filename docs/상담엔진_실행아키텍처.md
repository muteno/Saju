# 상담엔진 — 실행 아키텍처 (모델·지연·재미·페르소나 레이어)

> **위상: `상담엔진_설계.md`의 자매 문서.** 정본 = `docs/상담엔진_설계.md`(다른 세션·PR #28 · 알고리즘·정곡 랭킹·화술·정직성·자막 코퍼스 흡수). 이 문서는 **그 위에서 "어떻게 빠르게 서빙하고 게임으로 만드나"** = 실행(모델·지연·팬아웃·캐시) + 경험(재미 6축·예타 페르소나) 레이어만 다룬다.
> **충돌 회피 규약**: 이 문서는 알고리즘·정곡 선별·3단 어조·목소리 정의를 **재정의하지 않는다**(그건 설계.md 정본). 겹치면 설계.md가 이긴다. 여긴 서빙·모델·경험 축만.

## 0. 통합 프레임 — 「판은 계산, 대화는 LLM이 판 안에서」

운영자 정식화(260713): *"선택지는 항상 정해져 있고, 사용자는 판 안에서 논다. 거짓말이 아니라 그 판이 사주로 분석한 거니까. 사주는 마술이 아니라 로직이 확실한 학문이라, 로직만 잘 정립되면 먹힌다."*

이걸 아키텍처로 못박으면 **두 층 분리**:

| 층 | 무엇 | 누가 만드나 | 성질 |
|---|---|---|---|
| **판(board)** | 참인 정곡·유효 분기·유효 시기 = 유효 수(move)의 집합 | **엔진(`buildSaju`) = 결정론** | 진짜·검증가능 (설계.md §0-1·§3·§5 CALC) |
| **대화(play)** | 훅 문구·스타일 라우팅·답변지 보기·후속 가지 = 판 안의 실제 대국 | **LLM(Opus) = 지연 창에서 생성** | 살아있음·개인화, 단 **판에 종속** |

- **거짓말이 아닌 이유 = 팩트잠금이 곧 판 소속 검사**: LLM은 판에 있는 정곡만 말할 수 있다(설계.md §5 CALC·§10 화이트리스트). 판 밖 창작 = 폐기. 판이 실계산이라 대화가 아무리 유려해도 근거는 진짜.
- **마술이 아닌 이유 = 백스테이지가 실연산**(설계.md §6-1 마술사 스테이징): 겉은 "다 아는 듯" 연출이지만 뒤에선 진짜 트리를 계산·생성. 전지는 *연기가 아니라 사실*, 겸손은 *태도*.
- **로직이 먹히는 이유**: 판(엔진)의 명리 로직이 정립돼 있으면(설계.md가 담당), 그 위 대화는 로직을 포장만 바꿔 전달 → 운영자 검증("공부해봐서 앎")과 정합.

> 이 문서의 나머지(§1~§5)는 전부 **"판 안의 대화 트리를 지연 창 안에 어떻게 LLM으로 생성·서빙하나"**의 실행 설계다.

## 1. 모델 정책 — Opus 기본 · Sonnet 폴백 (운영자 확정 260713)

- **기본 = Opus 4.8**: "가능한 Opus 붙여 수준을 높인다." 상담 대화 생성의 주 모델.
- **폴백 = Sonnet 5**: "안 되면 / 일상적인 로직이면 Sonnet도 괜찮더라." 속도가 필요하거나 정형 로직인 구간.
- **다이얼 = 예타 `model × effort` 계승**(YETA §🤖): effort(low/mid/max)가 중간 노브. 파이프라인 모델 하향은 승인 축(기틀).
- **정책 한 줄**: **Opus = 숨은 창에서만**(오프닝·prefetch·깊은 풀이 = 사용자가 대기를 안 봄) / **Sonnet = 보이는 대기에서**(인라인 즉답·허당 quip). *사용자가 Opus 스피너를 절대 안 보게 하는 게 규칙.*

| 구간 | 모델 | 이유 |
|---|---|---|
| 오프닝 트리·깊은 풀이·prefetch | Opus 4.8 | 지연이 연출/선택시간에 가려짐 → 품질 최대 |
| prefetch 미완 시 인라인 즉답 | Sonnet 5 | 사용자가 지금 대기 → 속도 |
| 허당 quip·짧은 리액션·정형 전개 | Sonnet 5 × low | 일상 로직 = 저비용 충분 |

## 2. 지연 예산 = 트리 선(先)생성 (운영자 통찰 260713)

**VN 페이싱이 곧 지연 세탁기다.** 로딩 연출과 선택 시간을 트리 생성 예산으로 쓴다 = 프리페치 + 스펙셋(speculative). "분석 중"은 가짜 로딩이 아니라 진짜 Opus 연산(설계.md §6-1 백스테이지의 실행 형태).

| 구간 | 사용자 체감 | 뒤에서 도는 것 | 모델 |
|---|---|---|---|
| **진입 ~30s** | "사주 분석 중…" + 만세력 8장 플립·팔자 8글자 프로그레스(디제시스 연출) | 엔진 팔자 계산(<1s) → 정곡 랭킹(설계.md §3) → **Opus가 오프닝 단정 + 1~2레벨 대화 트리 생성** | Opus(전체 예산) |
| **노드 읽기·선택 ~1분** | 훅 읽고 답변지 보기 고민 | **Opus가 다음 노드 prefetch**(각 보기별 후속 서브트리) — 탭 순간 이미 warm = 즉시 | Opus(숨은 창) |
| **빠른 탭·prefetch 미완** | 즉답 기대 | 못 끝냄 | **Sonnet 인라인**(스냅) |
| **허당 quip** | 짧은 반응 | 죽은 시간 | Sonnet×low |

**정직한 한계**:
1. 30s는 *상한* — 오프닝은 폭우선(breadth-first) 1~2레벨만, 깊이는 선택 창에서 채움(30s에 전체 심층트리 다 못 뽑음).
2. prefetch를 (답변지 3~4개)×N레벨 전부 하면 분기 폭발 → **가장 유력한 보기 1~2개 · 1레벨만** prefetch(잘 랭크된 정곡은 특정 보기가 더 유력).
3. Opus가 15s에 끝나면 로딩 조기 종료 or 최소 드라마 시간까지 홀드(≥ 만세력 플립 연출 시간).
4. prefetch 헛방(예상 밖 분기) = 그 노드만 Sonnet 인라인 즉생성(버린 Opus 토큰 = 비용 손실 · 랭킹 좋을수록 적중률↑).

## 3. 런타임 팬아웃 — 사고 구역 분할 + Opus 취합 (운영자 발의 260713)

> *"시간을 단축시켜야 되면 사고 구역을 나눠서, 이걸 취합만 Opus가. 파악해야 되는 트리가 10~100이면 Opus 10~100이 그걸 (병렬로) 파악하고, 한 명 들어오는 대로 취합. 그러면 빨리 간다."*

**= 분신술을 런타임 추론 구조로.** 한 Opus가 트리를 순차 생성하면 30s 창을 넘길 때, **트리를 조각내 병렬 Opus 워커에 뿌리고(map), 취합 Opus 하나가 도착하는 대로 꿰맨다(reduce·스트리밍).**

```
board(정곡 후보 N개 · 유효 분기) = 엔진 결정론
        │  map: 분기/서브트리 단위로 분할
        ▼
  ┌─ Opus 워커 1 (정곡 #1 → 훅·전개·답변지·후속) ─┐
  ├─ Opus 워커 2 (정곡 #2 …)                     │  ← 병렬(10~100)
  ├─ …                                           │     스타일 헤더 공유(§6-2 라우팅)
  └─ Opus 워커 N                                 ─┘
        │  reduce: 도착 순 취합(스트리밍)
        ▼
  취합 Opus: 세로 고리 일관성·순서·중복 정리 → 최종 대화 트리
```

- **map 단위 = 정곡/서브트리**: 트리 노드가 서로 독립이라(각 정곡은 별개 근거) 병렬화가 자연스러움. 워커 = "이 정곡 하나를 판 안에서 대화 노드로 펼쳐라".
- **reduce = 스트리밍 취합**("한 명 들어오는 대로"): 먼저 온 워커부터 취합 Opus가 순서·세로고리(설계.md §2)·중복을 정리. 전원 대기(barrier) 안 하고 도착 순 처리 → 벽시계 = 가장 느린 한 워커 + 취합, N에 안 비례.
- **일관성 장치(품질 리스크 대응)**: 워커 전원이 **같은 스타일 헤더**(§6-2 청자 프로파일·목소리 설계.md §6-0)를 공유해 말투 이음새 방지 + 취합 Opus가 coherence 패스. 안 하면 N조각이 따로 놀아 "봉합선"이 보임.
- **비용·제약(정직)**: N Opus 동시 호출 = 비용 N배 + 레이트리밋. → **트리가 깊거나 창이 빡빡할 때만** 팬아웃(대부분 읽기는 순차 Opus로 충분). 팬아웃 폭 = board 상위 랭크 정곡으로 제한.
- **구현 대응물**: 이 map-reduce는 예타 dispatch가 아니라 **서버(Function) 안의 병렬 fetch + 스트리밍 취합** 또는 워크플로 팬아웃 패턴. 정적 SPA라 클라가 아니라 Function이 오케스트레이션(키·동시성 관리).

## 4. 대화 트리 선생성 — 지연 창에서 무엇을 짜나 (운영자 260713)

> *"그 구간에 PR#28하고, 내가 주는 에니어그램·대화법을 참고해서 **어떻게 대화할지도 미리 트리구조로** 짜놓는 거지."*

지연 창(§2)에서 Opus(필요시 팬아웃 §3)가 생성하는 것 = **판 안의 대화 트리**:
- **정곡 순서** (설계.md §1 5스텝 × 이중 트랙 · §3 랭킹) — 어떤 정곡을 언제 쏠지.
- **각 노드의 문구** — 훅→전개→확인 (설계.md §2 고리) · 3단 어조 태깅(§5 CALC/INFER/EVENT).
- **답변지 top-k 보기** (설계.md §2 답변지 메커니즘) — 계산된 후보(궁위·신살에서 도출)를 버튼으로 = **제약 디코딩**. 이게 "선택지는 항상 정해져 있다"의 실체.
- **스타일 라우팅** (설계.md §6-2) — 청자 프로파일(십신 분포·신강약·생왕고) + 운영자 제공 에니어그램·대화법으로 **같은 사실을 청자에 맞는 포장으로**. 내용 불변, 디코더 말투만 적응.
- **분기별 후속** — 각 보기 선택 시 이어질 서브트리(prefetch 대상).

**입력 3종**:
1. **엔진 board** = 정곡 후보·유효 분기·시기 (설계.md §3·§4).
2. **PR #28 목소리·화술 정본** = §6-0 공통 결 10·공용어 29 · §6 말맛 · §6-1 스테이징.
3. **운영자 제공 코퍼스** = 에니어그램(스타일 선택기 §6-2)·대화법·자막 흡수지도(`자막코퍼스_흡수지도.md`).

**제약 생성(constrained generation)**: LLM의 자유도는 board가 경계. 참인 정곡만 문구화(팩트잠금) · 유효 분기만 답변지화 · 흉살은 활용/질문형 변환(설계.md §5 가드). → **판 안에서 자유롭게 놀되 판 밖으론 못 나감 = 유려함 + 정직 양립.**

## 5. 캐시·리플레이 — 결정론 팔자의 배당금

- **캐시 키 = `hash(팔자 + 답변 경로)`**: 팔자가 결정론이라 같은 사람·같은 답 경로 = 같은 트리. 재방문·"아니야 되돌리기"(재미 §6 명식수 유령가지) = **warm 재사용 = 무과금·즉시**.
- **워밍 이월**: 지난 세션 트리를 캐시에 남기면 단골 재방문 시 오프닝 prefetch가 이미 데워짐(설계.md §6-1 전지 스탠스를 비용 0으로 강화).
- 예타의 R2 CAS 세션스토어는 불필요(단일 사용자·동시성 0) — localStorage + 서버 캐시로 충분.

## 6. 재미 레이어 6축 (경험 · 설계.md §7 템포 FX와 상보)

> 전부 무-LLM 결정론 완주(게임 시스템 축) — 상담 *대화*는 LLM(§1~§4), 게임 *시스템*(카드·나무·크리티컬)은 결정론. 킬러 한 줄: **"뽑기인데 돈이 아니라 생일이 뽑는다."**

| 축 | 핵심 | 재활용 자산 |
|---|---|---|
| 정곡 크리티컬 | 단정 일으킨 팔자 글자가 「的中」 인장으로 잠김 = 증거 시각화 | 팔자 프로그레스·契 인장 |
| 운명패 도감 | 정곡 확정마다 카드 발권·희소 %("인구 4%") | **12지+60갑자 카드 완성 = 조달비 0** |
| 명식수(命式樹) | 진행하며 나무 성장·유령가지=리플레이 | 오행색·수묵 에셋 |
| 호감도·대운 루트 | 츤 해금·10년 루트·daily 스트릭 | 호감도 게이지·표정 |
| 명함 공유카드 | 팔자+오행바+명대사+희소배지 = 바이럴 | 공유카드·마스코트 |
| 복채함 | 무료 후함 → 심화만 유료(츤 제안) | 복채 대사 |

**성역(무료 불변)**: 카드·크리티컬·명식수·기본 운세·공유카드. 유료는 오직 '깊이'.

## 7. 예타 페르소나 접목 (선택 · 분신3 감사)

- **두뇌 이식**: 카드 예시대화(말투 제1규칙)·tune 성향 다이얼(=츤레벨)·`[LV n]` 단골 해금·결정론 데일리 시드("오늘의 도사 기분").
- **몸통 폐기**: R2·Actions·dispatch·웜루프·4계정 폴오버 = 정적 사주앱엔 그 문제 자체가 없음.
- **반칙 이점**: 팔자 = 완벽한 결정론 기억 앵커 → 생일 한 줄만 저장, "기억" 90% 공짜.
- 도사 소수 로스터(츤 언니·근엄 노선비·젊은 무당)가 **같은 판을 각자 말맛으로** = 세컨드 오피니언 경제. 12지 마스코트 = 손님 정체성 토큰(도사 아님).
