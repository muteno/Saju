아이콘·픽토그램 정렬 기틀 이식

노뮤트에서 확립된 「정렬 이식팩」을 §8 절차로 Yeul에 심고, 현행 아이콘을 전수 실측한 감사 보고. — 결론부터: Yeul은 현재 아이콘 정렬 결함이 없다. 이 이식은 “커질 때”를 대비한 예방 가드레일이며, 실측이 캐낸 유일한 실질 개선점은 .scene-open(↗)의 우연 의존 정렬 한 건이다.

🧭 세 줄 요약 (처음 보는 분도)

  • 무엇을 했나 — “아이콘 버튼 줄 맞추기” 규칙 문서를 레포에 심고(docs/아이콘정렬_기틀.md), 마스터 규칙(CLAUDE.md)에 요약·포인터를 박았다. 그리고 지금 사이트의 모든 아이콘이 실제로 줄이 맞는지 컴퓨터로 픽셀을 재서 확인했다.
  • 왜 — 아이콘 정렬은 “한 번 맞춰도 다른 작업 하다 또 어긋나는” 대표적 재발 버그다. 개별로 고치지 말고 규칙으로 재발을 막자는 것.
  • 결과 — 지금 Yeul 아이콘은 다 정중앙(≤0.17px). 노뮤트를 괴롭히던 “리스트 팝업” 오정렬 패턴은 Yeul엔 아예 없다. 단 .scene-open ↗ 하나만 “자기 네모칸”이 없어 운에 기대 맞는 상태 → 튼튼하게 바꿀지는 버튼으로 여쭤봄.

① 현행 상태 재현 (헤드리스 실측 화면)

실제 css/styles.css 컴포넌트 클래스 + 005-ui-kit SVG를 그대로 옮긴 자기완결 하네스를 /opt/pw-browsers/chromium으로 렌더(서버·백그라운드 0). 아래 화면 하단의 RESULT_JSON이 각 아이콘의 셀중심 대비 편심(px) 실측값이다.

§5 실측 — 독립 아이콘 버튼 (셀중심 대비 편심)

  • 설치 ⤓ · 테마 🌙 · 닫기 X · fab ↑ · 소셜 IG · aud 🔋 = eccX 0.00px
  • 소셜 X = eccX 0.17px · 테마 🌙 = 0.015px (합격, <1px)
  • ↗ scene-open = eccX 0.00 / eccY −2.0*inline 라인박스 아티팩트, 화면 무영향

§2-3 방향성 아이콘 편심 (path 기하)

  • ↗ arrowUpRight · → arrow · ✕ close = 편심 0 (대칭 확인)
  • ▶ play M8 5v14l11-7z = +1.0px 우측 (삼각형 bbox 13.5)
  • 📄 draft(aud) = −0.56px 경계선 · 나머지 0

② 교차검증 — 분신 2기, 다른 렌즈로 수렴

프레임워크 §5·§6 정신대로 독립 Opus 분신 2기를 병렬로 돌려 기하(path)와 레이아웃(CSS) 두 각도에서 적대적으로 감사 → 같은 결론에 수렴.

분신 A · 기하 편심

모든 아이콘 path bbox를 직접 계산 → play ▶ +1.0px가 유일하게 의미 있는 편심(전형적 재생삼각형 스큐), draft −0.56px 경계선. arrow·arrowUpRight·close는 삐뚤어 보여도 기하적으로 완벽 대칭.

분신 B · §4 정렬 감사

독립 아이콘 버튼 전부 고정 사각형 + place-items:center로 중앙 강제. 유일 예외 = .scene-open(자기 박스 없음 → 인라인 크기·26px 패딩 우연에 의존). “리스트 팝업 헤더X↔행버튼 세로열” 패턴은 Yeul에 없음 → N/A, 없는 위반 안 만듦.

⚠ 리스크 / 한계

🔧 권고 — 유일한 실질 개선점 (승인 시 30초 적용)

.scene-open에 다른 아이콘 버튼처럼 중앙 정렬을 셀에 강제: .scene-open{ display:inline-grid; place-items:center; line-height:0; } — ↗ 위치는 시각상 불변(no-op)이나, 인라인 크기·패딩 우연 의존을 없애 회귀 취약점을 제거. eccY −2px 아티팩트도 사라짐. UI 변화 없음 → 이 커밋엔 미포함, 버튼으로 여쭤봄.

⑤ 재현 한계 각주(§5-5) — ① 측정은 실제 페이지가 아니라 정렬 관련 CSS·SVG를 그대로 옮긴 자기완결 하네스(file://, 서버·백그라운드 프로세스 0). 정렬 기하는 정적 마크업이라 동일하나, 실폰트 로드·리빌 모션·DPR 스케일은 배제. ② DPR 1 기준. ③ getBBox는 Chrome 채움 기하 bbox(스트로크 미포함) — 편심(비대칭)만 본다. ④ .scene-open eccY는 inline <span> 라인박스 영향(아이콘 결함 아님). ⑤ 검증 = node --check 전 js 통과 · 헤드리스 렌더 JS 에러 0 · 좌표 실측 + 스크린샷 육안 확인.
