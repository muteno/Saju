2뭐가 생겼나 (구성)

  • sw.js — 서비스 워커(새 파일). 사이트와 브라우저 사이에 서는 작은 집배원: 온라인이면 항상 서버의 최신을 가져오고(캐시 눌어붙음 사고 원천 차단), 끊기면 담아둔 사본으로 화면을 연다. 첫 화면 셸 20개 파일을 설치 순간 미리 저장.
  • 설치 버튼 — index.html 내비 + js/app.js 배선. 브라우저가 "설치 가능" 신호를 줄 때만 다운로드 아이콘이 등장(기존 .icon-btn 원형 버튼 계승 = 새 디자인 값 0). 누르면 설치 창, 끝나면 토스트로 알림.
  • manifest.webmanifest 보강. 앱 이름("어휴게소")·아이콘은 있던 그대로, 설치 식별자와 원형 아이콘(maskable) 대응만 추가.
  • FAQ 한 줄 추가(002-brand). 설치 버튼이 안 뜨는 아이폰 사파리용 경로 안내 — "탈출구는 가까이 둘수록 좋으니까요."
  • CSS 한 줄. [hidden]{display:none!important} — 숨김 속성이 원형 버튼 스타일을 이기게 하는 안전핀.

3실측 스크린샷

4장단점 · 리스크

장점

  • 홍보 사이트가 폰 홈 화면·PC 바탕화면에 영화 앱처럼 앉는다 — 재방문 마찰 0.
  • 지하철·비행기 모드에서도 열림(첫 방문 때 사본 저장).
  • 네트워크 우선 전략이라 배포 즉시 최신 반영 — 캐시 때문에 옛 화면이 남는 사고 없음.

리스크 · 관리 포인트

  • ⚠ PRECACHE 목록 유지보수 — 새 js/css/핵심 이미지를 추가하면 sw.js 목록에도 한 줄 추가해야 오프라인 첫 화면이 완전하다. (CLAUDE.md 작업 메모에 규칙 명문화)
  • 폰트는 CDN(외부)이라 오프라인에선 시스템 폰트로 폴백 — 읽기엔 문제없고, 모양만 살짝 다름.
  • 효율/품질 저하 없음 — 서비스 워커는 백그라운드 파일 1개(3KB), 페이지 로딩 경로는 그대로.

6재현 한계 각주

  • 헤드리스 브라우저는 설치 신호(beforeinstallprompt)를 쏘지 않는다 — 버튼은 강제 노출로 스타일만 검증했고, 실제 등장·설치 흐름은 배포 후 실기기(크롬/엣지)에서 확인 필요.
  • 검증 환경이 폰트 CDN을 막아 스크린샷은 시스템 폰트 폴백 상태 — 실서비스 온라인에선 Pretendard/Jakarta로 뜬다.
  • 아이폰 사파리는 설치 신호 자체가 없는 플랫폼 — 버튼 대신 FAQ 안내(공유 → 홈 화면에 추가)로 커버.
