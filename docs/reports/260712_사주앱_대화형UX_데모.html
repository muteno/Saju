재현 한계 각주 (정직 모드)

  • 만세력 = 데모용 고정값. 생일을 바꿔도 팔자가 같음 — 실앱은 만세력 계산 모듈(라이브러리/절기표) 필요. 데모 팔자는 대사와 정합(寅=역마 · 子·午=도화 · 金 0=금약)하도록 설계.
  • 일러스트 = CSS 플레이스홀더. 확정 웹툰 이미지(드래프트 §4 프롬프트)로 교체하는 자리. 표정 5종 슬롯(기본/츤/놀람/미소/허당)만 먼저 배선해 둠.
  • 대사 = 무-LLM 고정 스크립트. 실앱 = 템플릿 사다리 + LLM 승격 레이어(드래프트 §3). 선택지 분기 구조는 실제 설계와 동일.
  • 저장/공유 버튼 = 연출만. 이미지 내보내기는 실앱 구현 항목.
  • BGM·햅틱 없음. 햅틱은 시각 펄스로 흉내(설정).
  • 옵션 규모 축소 사유: 확립 기틀이 없는 신규 앱 컨셉 데모 = 규약상 '단순 대상' 취급. 색은 폐쇄 팔레트 3종만(자유 hex 없음).
  • ?fast=1 파라미터 = 검증용 고속 모드(타이핑 즉시·자동 진행).
