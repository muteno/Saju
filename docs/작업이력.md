## 260706 — PWA 설치: 웹을 앱처럼 (설치 버튼 + 오프라인)

- **요지**: 사이트를 브라우저에서 "앱 설치" → PC 바탕화면·폰 홈 화면 바로가기 + 오프라인 열람 가능하게. 신규 `sw.js`(서비스 워커) · 내비 설치 버튼(`index.html`+`app.js`) · `manifest.webmanifest` 보강(id/scope/lang/maskable) · FAQ 아이폰 안내 1건(`002-brand`) · CSS `[hidden]` 가드 1줄.
- **전/후**: 전 = 매니페스트·아이콘만 있고 설치 동선·오프라인 없음 → 후 = 설치 가능 신호 시 내비에 ⤓ 버튼 등장, 첫 방문 때 셸 20파일 캐시로 오프라인 완전 렌더.
- **결정근거**: 캐시 전략 = **네트워크 우선**(온라인이면 항상 최신 → 정적 사이트 최다 사고인 "옛 화면 눌어붙음" 원천 차단, 오프라인만 캐시 폴백). 설치 버튼은 기존 `.icon-btn` 계승(신규 디자인 값 0 = 플레이그라운드 불요 판단). 기틀 교차검증 미발동 — js/core·knowledge 구조/계약 무변경(FAQ는 콘텐츠 데이터 = 주변부, app.js는 부트스트랩).
- **검증(실기동)**: `node --check` 전 파일 통과 · 매니페스트 JSON 파스 OK · 헤드리스 크로미움 실측 — SW 등록·활성·페이지 제어 ✓, **서버 프로세스 실제 종료 후** 새로고침 = 전체 렌더 ✓(포스터 포함), 오프라인 딥 경로(`/no-such-path`) = index 폴백 200 ✓, 페이지 JS 에러 0 ✓, 설치 버튼 42px 원형 계승 스크린샷 ✓. 상세 = `docs/reports/260706_PWA설치.html`.
- **관리 포인트(다음 세션 주의)**: 새 js/css/핵심 이미지 추가 시 `sw.js` PRECACHE 목록에도 추가(CLAUDE.md 작업 메모 명문화). 헤드리스는 `beforeinstallprompt` 미발화라 실제 설치 UX는 배포 후 실기기 확인 필요.
