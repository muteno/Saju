# 제1부 — 「전」: 파도 이전의 앱은 어떤 상태였나

## 1. 총평 — "껍데기는 상용, 속은 목업"

파도 이전의 앱은 화면 4개(홈 → 입력 → 로딩 → 결과)의 미연시풍 외형과 검증된 만세력 엔진(포스텔러 픽스처 19테스트 통과), 증류 KB(일주 60/60 + 인상 19키, 인용 223건 축자 검증)라는 **훌륭한 뼈대**를 갖추고 있었다. 그러나 화면과 데이터 사이의 배선이 절반쯤에서 끊겨 있었다. 구체적으로: 화면에 보이는 것의 상당수가 하드코딩 목업이었고, 엔진이 이미 계산해 놓은 자산의 상당량이 화면에 도달하지 못했으며, 사용자가 틀린 값을 넣으면 앱이 침묵한 채 남의 사주를 보여줬다. 아래는 축별 전수 기록이다.

## 2. 데이터 진실성 — 목업·하드코딩 전수

- **이름**: 홈 상단 대문짝 이름이 고정 문자열 'uibowl'. 누가 접속해도 uibowl.
- **날짜**: 홈 날짜가 'JUL 16' 고정. 실제 오늘과 무관. 오늘의 일진도 `engineToday(2026, 7, 16)` — 2026년 7월 16일로 하드코딩되어 매일 같은 일진.
- **오늘의 운세 점수**: 82점 고정. 코드 주석에 "점수/문안은 서비스 정책값(placeholder)"라고 명시된 상태로 라이브에 노출. 문안("차분한 정리의 날")도 고정.
- **사주 원국**: 홈·결과 모두 기본 표시가 샘플 프로필(1990-01-01 08:24 여성)의 계산 결과. `mockPillars`/`mockOhaeng`이 모듈 로드 시점에 고정 계산됨.
- **상태바 시계**: '9:41'/'8:58' 등 화면별 고정 문자열.
- **홈 요약("한눈에 보기")**: "병화(丙火) 일간, 태양처럼 뻗는 기운…" — 샘플 프로필에 대한 고정 문안 2줄. 사용자가 누구든 병화 일간 요약.
- **세운 연도**: `buildReading(input, unseYearName = '병오')` — 2026년이 하드코딩. 해가 바뀌어도 병오년 운세.
- **로딩 연출 타일**: 경·병·계·무 4글자 고정(누구의 사주와도 무관).

## 3. 입력 신뢰성 — 조용한 실패의 계보

- **자유 텍스트 입력**: 생년월일시를 "1990/01/01 08:24" 형식의 문자열 하나로 받고 정규식 분해. 형식이 조금이라도 어긋나면?
- **조용한 샘플 폴백(치명)**: 파싱 실패 시 에러 없이 `state: undefined`로 결과 화면 이동 → 결과 화면은 "없으면 샘플" 폴백 → **사용자는 자기 사주인 줄 알고 남(샘플)의 사주 풀이를 읽는다.** 점술 서비스에서 최악의 무결성 사고 유형.
- **음력 셀렉트**: '음력'·'음력(윤달)' 옵션이 버젓이 존재하지만 엔진에 음력→양력 변환 코드가 0줄(260718 전수 grep 단정). 음력 선택 후 제출하면 음력 날짜가 양력으로 계산됨 = 총체적으로 틀린 사주.
- **출생 도시**: '충청남도 공주, 대한민국' 고정 readOnly 필드 + 장식 돋보기. 엔진은 `longitude` 파라미터로 진태양시 보정을 지원하는데(서울 -32분) 어댑터가 전달하지 않아 전원 서울 기준.
- **시간 모름**: "◻ 시간 모름" 텍스트가 있으나 클릭 불가 장식.
- **보정값 칩**: "조후와 궁성 보정"·"합에 따른 오행 변화" — 엔진에 대응 기능이 없는 순수 장식(항상 ✓ 켜짐으로 그려짐). 반면 엔진이 실제 보유한 옵션(진태양시 on/off, 야자시 유파)은 노출 경로 없음.
- **검증**: 1899년, 13월, 2월 30일 — 무엇을 넣어도 통과(파싱만 되면). 프로필 저장 자체가 없어 재방문 시 백지부터.

## 4. 결과 화면 — 엔진 산출의 3할만 도달

L3 리포트는 항상 10섹션(원국표·구조판정·일주·일간·일지·십신·합충·신살·대운·세운)을 꽉 채워 반환하는데(260718 실덤프: 3케이스 전 섹션 유효, keyset 적중률 83~86%), 화면 어댑터(`toReading`)는 그중 **3섹션만** 소비했다:

- 노출: 구조판정 lines, 일주 증류(핵심+성격 3+주의 2), 세운 발췌 3문단.
- 미노출: **일주 증류의 직업·관계·물상·관점차이·인용**(증류본에서 가장 공들인 층), **십신 구성**(distribution+상위 발췌), **합충 관계**(이름·위치 lines + 종류별 발췌), **신살 8~13종**(케이스별), **일간·일지 발췌**, **대운 표 10기**(엔진 계산 완료 상태로 미사용), **지장간**(원국표 행 자체 부재).
- **세운 본문 치명 결함**: KB 웹 번들이 전 유닛을 앞 8문단으로 절단하는데, 세운 문서는 앞 12문단이 인사말+목차 구조라 **라이브의 '올해의 운' 섹션에 "‑ 글의 차례 ‑ / 1. 병오년의 의미 / 2. …" 목차만 표시**되고 있었다(전체 373문단 중 실본문 0). 즉 상용 관점 최핵심 콘텐츠(올해 운세)가 빈 껍데기.
- **딥링크/새로고침**: 결과 화면이 `location.state`에만 의존 — 새로고침하면 상태 소실 → 샘플 폴백. URL 공유 불가(받는 사람은 무조건 샘플). 공유 버튼 자체도 없음.
- 결과 화면은 스크롤 없는 1화면 고정 — 정보량을 늘릴 그릇 자체가 없었음.

## 5. L4 도사 대화 — 설계만 존재

- 서술 표준 6원칙(사용자 확정)과 대화형 설계 플레이그라운드(대분류 5×소분류 10, 컨트롤 50개 — Q.06 머지)는 완성. 그러나 앱에는 대화 기능 0줄: Pages Function 없음, functions/ 디렉터리 없음, 주제 분류표 없음, 타이프라이터 없음.
- 결과 화면의 VN 컨트롤 3칩(넘기기/자동 대화/지난 대화)은 onClick 없는 장식 — 상용 VN의 필수 기능 문법을 하고 있어 "고장난 앱"으로 오인되는 부류.

## 6. 재방문·성장 루프 — 0

- 프로필 저장 없음(localStorage 어디에도 사주 데이터 없음 — 다크모드 설정만 저장). 재방문 = 백지.
- 공유 수단 없음(버튼·URL·OG 어느 것도). 카톡에 붙일 것이 없음.
- PWA 없음: manifest 없음, 설치 불가, 아이콘 PNG 없음, OG 메타 없음(카톡 미리보기 = 제목 한 줄), description 메타도 없음.
- 하단 내비 5탭 중 홈 제외 4탭이 사실상 미완: 콘텐츠/마이 = 완전 무배선인데 **마이 탭에는 하드코딩 빨간 알림 점**(거짓 배지), 오늘/분석 = 동일한 목적지로 중복.

## 7. 게이트·파이프라인의 숨은 결함 2건

- **verify 순서 버그**: 1게이트(check_refs)가 CLAUDE.md 참조 경로 실존을 검사하는데 `dist/`는 5게이트(빌드)가 만드는 생성물 — **새 컨테이너 첫 verify는 구조적으로 무조건 실패**(2회차부터 통과). "게이트 빨강 = 커밋 금지" 규범을 새 세션마다 오작동시키는 신뢰 훼손 축.
- **세운 KB 절단**(4절에 상술): build_kb.mjs의 BODY_PARAS=8 전역 절단.

# 제2부 — 「후」: 두 파도가 끝난 지금의 앱

## 1. 총평 — "실계산·실저장·공유 가능·근거 병기, 그리고 열 수 있는 대화"

지금의 앱은 접속 순간부터 결과 공유까지 전 구간이 실데이터로 돈다. 파도 1(10커밋, PR #60)이 데이터 종단을 뚫었고, 파도 2(2커밋, PR #62)가 15인 실측으로 잡은 UI·무결성·접근성 결함을 일괄 수선했다. 전 과정 verify 6게이트(기틀 경로→vendor 드리프트→엔진 19테스트→증류 검증+시험은행 리플레이→프로덕션 빌드→실브라우저 스모크) 그린 상태로만 머지했다.

## 2. 사용자 여정 기준의 '후'

**첫 방문**: 온보딩 홈(가짜 수치 0 — 오늘 날짜·일진만 실계산) → "내 사주 시작하기" → 입력 화면.

**입력**: 년/월/일/시/분 구조화 필드. 숫자만 수용(문자·초과 자릿수 즉시 필터), 년 4자리를 치면 월로, 월을 채우면 일로 **자동 포커스 체인**(데이트피커 없이 '타이핑 피커' 체감). 각 필드에 년·월·일·시·분 단위 어도먼트. 검증은 명시적: 1899년→"1900~2100년만 지원해요", 2월 30일→"실제로 있는 날짜인지 확인해 주세요" — 에러는 **제출 버튼 위 고정 푸터**에 떠서 폴드 아래로 숨지 않고, 틀린 필드는 테두리가 붉게(MUI error) 표시된다. 음력은 '준비 중' 정직 비활성(흉내 금지). 도시 26개 셀렉트 → 경도가 엔진 진태양시 보정에 실전달(순천 선택 시 -30분 실측). 시간 모름 토글(히트영역 43px 확장) — 켜면 정오 대입(일주 오판 불가능 실증: 보정 포함 경계까지 최소 10.5시간) + 시주 의존 산출물 잠금. 보정 칩은 실기능(진태양시 on/off·야자시 유파). 제출 = 프로필 저장(localStorage 다중 — 이름+생일 동일 시 갱신) → 재방문 시 "저장된 사주" 칩 행에서 탭 한 번으로 불러오기.

**홈(내 사주)**: 오늘 실날짜(KST) + 내 이름 + 말풍선에 오늘 일진과 관계 문안. **오늘의 운세 점수 = 엔진 diaryDayInfo(오늘 일진 vs 내 원국의 합·충·형·파·해·원진·공망 발동) 기반 결정론 정책 매핑**(base 75, 사건별 가중, clamp 45~95 — rubric은 코드에 명문화) + 점수 아래 근거줄("내 월주와 사해충" 식) 병기. 십신 테마 라벨("경쟁심이 커지는 날") 동반. 원국 7행 표 + 오행 분포(바 길이 = 실비율, "2개 · 발달" 결합 표기). 한눈에 보기 = 내 일주 특성·올해 실문안. 액션 버튼 3개 전부 실기능(리포트/오늘운세 공유/정보수정 — 라벨 캡션 부착).

**결과(도사의 방)**: 첫 화면 = VN(스테이지 + '‹ 홈' 복귀 칩 + 7행 원국표[십성·천간·지지·십성·**지장간**·십이운성·**12신살**] 중앙 정렬 + 진태양시 캡션 칩["진태양시 07:30 (보정 -30분 · 순천)"] + 불투명 오행 스트립 + 도사 한마디[특성+올해 압축 2섹션] + 공유 원형 버튼). 시트가 48px 미리 보여 "아래 더 있음"이 구조로 전달됨. 스크롤하면 리포트 시트: **원국 구조 판정 → 일주 이야기(핵심·성격·직업·관계·주의·물상·관점차이 — 문헌 이견 병기, 접기 지원) → 십신 구성(보유 십신 칩 + 상위 발췌) → 합충 관계 → 신살(발췌 5종 + 나머지 칩) → 올해의 운(실본문 — 목차 아님) → 대운 레일**(간·지 모두 오행 타일, 나이·연도, 현재 대운 헤더 명명 "지금 경신 대운" + 하이라이트 + 에지 페이드 + "좌우로 넘겨 보세요"). 출처는 카드 푸터 1줄로 집약(문서명만). 발췌는 정제기를 통과한 실내용만(목차·인사말·SEO·남의 사건 사주풀이 차단).

**도사 대화(L4)**: "아이샤에게 물어보기" — 고정 5주제(성격·올해·직업·관계·주의) 선택지(플레이그라운드 정본 규격) → 타이프라이터 대사(28ms/자, **탭=즉시 스킵 — 타이머 미정리 버그 수선**), 진행 표지(3/10)·주제 복귀 제공, 대사마다 "근거 보기" 접힘(출처 문헌). 서버에 `ANTHROPIC_API_KEY`만 등록하면 같은 대사가 Claude(L4 서술 표준 6원칙 시스템 프롬프트 + 근거 밖 주장 금지) 서술로 승격되고, 없으면·실패하면 조용히 코퍼스 원문 폴백 = **완결 동작**.

**공유(성장 루프)**: 결과가 URL이다(`/result?y=…&t=…&city=…&n=…`) — 새로고침·카톡 전달 안전. 보내는 쪽: 리포트 하단 + 대화박스 헤더 + 홈 오늘운세 3개 진입점(Web Share, 폴백 = 클립보드 + "카톡에 붙여넣기" 안내). **받는 쪽: "공유받은 사주 — ○○님의 리포트" 배너 + "내 사주도 풀어보기" CTA** — 파도 2 이전에는 수신자가 남의 리포트를 자기 것처럼 보고 전환 경로가 0이었다(바이럴 루프 물리적 절단 → 봉합). 카톡 미리보기 = OG 이미지(1200×630)·제목·설명. 깨진 링크는 "공유 링크가 잘못됐어요" 명시 에러(남의 샘플 무언 노출 제거).

**설치·외피**: manifest + 아이콘(192/512/애플터치 — 기존 파비콘 렌더, 신규 디자인 창작 없음) + theme-color 라이트/다크(수동 토글에도 동조) + description. 핀치 줌 복원.

## 3. 무결성·정확성 축의 '후'

- 불가능 날짜(2/31) = 입력·딥링크 양쪽 차단. 오염 프로필(연도 비정수 등) = 에러 화면. 보정분 원시 float(0.13333…) = 반올림.
