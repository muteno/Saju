# 상담 엔진 설계 — 정곡·고리 알고리즘 (정본 · 260713)

> 목표: 만세력 엔진의 **실계산 결과**를 "진짜 상담받는 맛"으로 전개한다 — 오! 하는 정곡, 하나 맞추면 다음으로 파고드는 고리고리.
> 근거: 분신 4기 병렬 아이데이션(유튜브 자막 308편 화술 / 실전 상담사례 76편 구조 / 정곡 랭킹 알고리즘 / 고리 상태기계) 수렴.
> 원칙: **컨셉·페르소나 없음. 정확도 우선. 원 소스 중심.** 없는 것을 있다고 하지 않는다.

## 0. 한 줄 요약

```
buildSaju(생년월일시,성별) → 정곡 후보 산출(계산이 참인 것만) → 임팩트 랭킹
→ 5스텝 골격에 스텝당 1발 배치 → 고리(훅→전개→확인→분기→다리) 체인으로 전개
→ 엔딩 = 용신 처방 + 시기 + 격려
```

## 0-1. 동작 원리 — 만세력 = 프롬프트, 상담 = 자기회귀 트리 샘플링 (운영자 정식화 · 260713)

> "처음엔 아무 정보 없는 데서 만세력만 읽고 시작하고, 거기서 트리 구조로 간다.
> LLM 모델이 단어를 고르는 것과 완전 일치한다. 그걸 재밌게 표현 방법만 바꾸는 것."

| LLM | 상담 엔진 |
|---|---|
| 프롬프트(초기 컨텍스트) | 만세력 계산 결과(buildSaju) — 사전 정보 0에서 시작 |
| 어휘(vocabulary) | 정곡 후보 풀(계산이 참인 것만 · §3) |
| 로짓/확률 | salience 점수(희소×체감×안전) |
| 다음 토큰 샘플링 | 다음 고리 선택 |
| 컨텍스트 갱신 → 재계산 | 사용자 답(맞아/아니야)이 상태에 들어가 다음 선택 재가중 |
| 가지치기 | "아니야" → 해당 시드 계열 prune |
| **사후 갱신(추가 규칙)** | **"맞아" → 인접(같은 축) 후보 가중 상향(boost)** — prune의 대칭 |
| 온도(temperature) | 템포 규칙(§7) + **trust 어닐링**: 적중 누적↑ → 어조 단정 쪽, 빗나감 → 질문 쪽 |
| 디코더(표현) | 표현층(slot) — 지금은 템플릿, 후속 LLM 스왑. 구조는 불변 |

핵심 성질 = **자기회귀**: 고정 대본이 아니라 매 답이 다음 선택의 분포를 바꾸는 샘플러. 실제 술사가 답을 들으며 방향을 트는 것과 동형이라 "살아있는 상담" 느낌이 난다. 추가 데이터(아카이브·자막)는 어휘의 해석 사전을 두껍게 하는 살(肉)이고, 뼈대는 선별기(확률)+그래프(트리)+표현층(스킨).

## 1. 골격 — 5스텝 × 이중 트랙

**정석 골격(운영자 해설책 '읽는 길' 계승):**
①글자(일간·팔자·오행) → ②관계(십신·합충) → ③자리·흐름(궁위·운성·신살·공망) → ④종합(신강약→격국→용신) → ⑤시간(대운·세운)

**이중 트랙(실전 상담 76편 실증):** 재미있는 통변은 8~9할이 특이점부터 친다. → **정석 골격을 유지하되, 계산에서 잡힌 최고 임팩트 특이점을 '하이라이트 카드'로 선행 노출**하고 그 특이점의 세로 고리가 5스텝을 관통한다.

**개시 고정(0스텝):** 일간(체) 확정 + 월령(계절/조후) — 모든 풀이의 좌표원점.
**엔딩 고정:** 길흉 판정으로 끝내지 않는다. **용신 기반 실천 처방 + 다가오는 대운·세운 타이밍 + 격려 한 줄.**

## 2. 고리(Ring) — 대화의 원자 단위

```
고리 1개 = 훅(계산 사실 한 방) → 전개(성향·삶 해석) → 확인(맞아/아니야/글쎄)
         → 분기(맞아=파고들기 / 글쎄=완화 / 아니야=가지치기+정직 보정)
         → 다리(다음 고리 예고)
```

- **세로 고리** = 한 특징이 5스텝 관통. 예: 金0(①) → 식상 부재(②) → 설 자리 없음(③) → **용신이 바로 그 빈 金(④·큰 정곡)** → 金 들어오는 N세 대운(⑤).
- **가로 고리** = 같은 스텝 안 확장(형제 십신, 이웃 궁위).
- **"아니야" = 해석 분기 스위치**(게임 장치 아님): 실전 정석 — *"종격 사주는 살아온 이력과 대조해 맞지 않으면 정격으로 해석하라."* 사용자의 답이 용신·조언 트리를 실제로 갈라친다. 원칙: *"사주는 반드시 그 당사자에게 물어보면서 해석해 나가야 돼요."*
- **답변지 메커니즘(운영자 정식화)**: 정곡이 꽂힌 순간("억?"), 맞아/아니야 대신 **계산된 후보 답안 3~4개 + [아니야]를 보기로 제시**. 보기 자체가 계산 산물 — 충 맞은 궁이 관장하는 삶 영역(궁위론)에서 도출(예: 일지 충+역마 → [이직·일] [이사·이동] [관계 정리]). 어떤 보기를 골라도 후속 가지가 사전 준비돼 있고, 고른 답은 인접 후보 boost의 자기회귀 입력이 된다. LLM 대응물 = **top-k 제약 디코딩**(자유 입력 대신 계산된 후보를 버튼으로).
- 문구(slot)와 근거(엔진 필드 술어)를 **분리** — 근거는 재현·검증 가능, 문구는 후속 LLM 승격 슬롯.

## 3. 정곡 후보 + 랭킹

**후보군(엔진 필드 → 판정 조건 → 통설 체감, 전수는 분신C 보고):**
- A군 글자: 오행 0개/4개+/편중≥5(종격 후보), 십신 그룹 과다·전무(합=7이라 단일 십신 0개는 흔함 → **그룹 단위 판정**)
- B군 관계: 충·삼합 완성(희소 최상)·반합·원진·형·간여지동·귀문†
- C군 신살·일주: 역마·도화·화개(안전 최상=폴백 주력), 괴강†·백호†·양인†(60갑자 룩업)
- D군 자리: 공망 궁, 배우자궁(일지) 충/원진/공망, 사회궁(월지) 왕지, 근묘화실
- E군 종합·시간: 신강약(정곡감 낮음—**맥락용 본식**), 패턴(재다신약 등), 대운 교운기, **운 충합 완성 시기(체감 최강)**
- († = 엔진 미계산 → 신규 지식모듈 `014-special-sinsal`로 60갑자·지지쌍 룩업만 추가하면 계산 가능. 창작 아님.)

**임팩트 = 희소성×0.35 + 체감성×0.40 + 안전성×0.25** (체감성: 성향<사건<시기특정)
**배치: 스텝당 대표 정곡 1발** + 보강 1개까지. 신살은 양념, 종합(강약·용신)이 본식 — 단편이 종합을 뒤집지 못한다.
**폴백(중화·밋밋 사주):** 살을 창작하지 않는다. 일간 물상 + 조후 + 최강 relation + 근묘화실 서사 + "균형 자체가 특징" 정직 프레이밍 + 교운기. 확인 빈도 낮추고 연출 비중 높임.

## 4. 시기 정곡 (하이라이트)

- 대운 교운기: `|현재나이 − daeun[k].age| ≤ 1` → "지금이 판 바뀌는 교운기".
- 대운 지지 × 원국 4지지에 `branchRelations()`(008 재사용) → 충/원진/형/합 걸리는 10년 창.
- 세운을 과거로 확장해 **삼형·충 완성 연도** 특정 → "○○년(○살) 무렵 큰 변화 있지 않았어?"
- 랭킹 = 궁 중요도(일지>월지>시지>년지) × 관계 강도(충>원진>형>합) × 현재 근접성.

## 5. 정직성 3단 어조 (콜드리딩과의 경계선)

| 유형 | 정의 | 어조 |
|---|---|---|
| CALC | 계산 사실 | **단정 OK** — "네 팔자에 金이 0이야" |
| INFER | 통설 해석 | **hedge 강제 + 참고 라벨** — "~한 경향이 있을 수 있어" |
| EVENT | 시기 사건 | **질문형만** — "그 무렵 변화 있지 않았어?" (과거 단정 금지) |

가드: ① 조건 `===true`인 것만 발화(없는 것 언급 금지) ② 흉살(백호·귀문·형)은 잠재·활용형/질문형으로 변환 — 사고·질병·이별·사망 단정 금지 ③ 결핍은 CALC로 말하되 부정 서사 금지 → 보완운으로 연결 ④ "무조건 좋다/나쁘다" 렌더 금지(명·암 병기).

## 6-0. 목소리 정의 — 크로스-스피커 공통 뉘앙스 (자막 코퍼스 4채널 교차 검증 · 260713)

> 초코서당 114·도화도르 61·길 8·포스텔러 6편 정독 결과, 화자 전원이 공유하는 결 10개 확인.
> 최강 2개(코퍼스 빈도): **"~볼 수도 있다" 가능성 화법 881회** · **"활용"(흉→쓰임 전환) 489회.**

- **공통 결 10**: ①가능성 화법 ②흉→활용 전환(반드시 세트) ③사주=구조론(운명론 아님) ④위로 한 스푼 ⑤전문어→즉시 일상 비유 ⑥"결국 선택은 너" 주체성 클로징 ⑦미신·겁주기 비판 ⑧좋다/나쁘다 이분법 거부(양면 병기) ⑨실용 개운 마무리 ⑩따뜻한 사제 톤.
- **배합 공식**: 초코서당의 분석적 정직(뼈대) + 도화도르의 쉬운 번역·셀프리딩 자립(살) + 길의 겸손 유보(안전장치) + 포스텔러의 위트(양념 소량). — 운영자 컬렉션 편향(이론·구조 지향, 반운명론) 실측 근거.
- **전지 스탠스 조정(§6-1 보정)**: 속은 전지(전체 트리 사전 계산 = 사실), **겉은 겸손** — "다 안다" 말투 ✗, "구조는 이렇게 읽히고, 나머지는 네 몫" ✓. 코퍼스 공통 결(겸손·유보)과 충돌하지 않는 유일 해법 = **겸손한 전지**.
- **금지(화자 전원 명시 거부)**: 단정적 예언(사망·이혼·합격 시점) · 신살 공포마케팅 · 부적류 개운 강요 · 전문어 나열 압도 · 세일즈 톤.
- **공용어 사전(29어)**: 기운·뿌리·발동·활용·감당·그릇·풀리다·타고나다·치우치다·흐름·용신·개운·과다/발달/고립 등 — 분신 보고 원문 참조(표현층 어휘 원료).

## 6. 말맛 (유튜브 308편 화술 상수)

- **훅 공식**: 통념 꺾기 — "신강=기 센 사람? 오해야", "사주로 부의 크기는 몰라".
- **정곡 강도 = 계산 확신도 연동**: 뚜렷하면 단정 / 조건부면 "아마 ~했을 거야" / 애매하면 확인질문으로 분기(실전 0295 방식).
- **접속 슬롯**: 근거="왜냐면" → 전환="그런데 여기서 중요한 게" → 처방="그럼 어떻게 하냐" → 착지=본질/타이밍.
- **비유 상수(툴팁)**: 신강약=인풋/아웃풋 · 용신=마법지팡이 아닌 평생 방향 · 오행=계절 에너지 · 재다신약=부잣집의 가난한 머슴 · 대운=연극 막 전환 · 신살=상시 아닌 우발 발동.
- **드릴다운 사슬**: 특징→성향→삶영역(직업·연애·재물·건강)→시기→조언.

## 6-1. 전지(全知) 스탠스 — 마술사 스테이징 (운영자 정식화 · 260713)

> "마술 부릴 때 연기하면서 뒤에서는 다 하고 있는 것처럼, 다 알고 있다는 듯."

- **백스테이지/프론트스테이지 분리**: buildSaju가 전체 트리(모든 가지·모든 시기)를 **한 방에 사전 계산** — 사용자가 어떤 보기를 골라도 즉시 이어지는 건 연기가 아니라 사실. 프론트는 그걸 **하나씩만 꺼내 보여주는 연출**.
- **스탠스 3요소** (캐릭터 스킨 아님 — 화자 태도. 스킨(꼬미 등)은 별도 후속):
  1. 이미 다 본 사람의 침착함 — "계산 중…" 후 답이 바뀌는 모습을 절대 노출하지 않는다.
  2. 리딩형 화법 — "있었나요?" ✗ → "**있었지?**" ✓. 확인 질문도 몰라서 묻는 게 아니라 도장 찍는 태도.
  3. 빗나가도 안 당황 — 우아한 prune. **단, "사실 그것도 맞다" 식 재해석(무오류 우기기) 금지** — 그 순간 콜드리딩이 되고 §5 정직성이 무너진다. 전지함은 '당황하지 않는 태도'로 구현하고, 틀린 가지는 틀렸다고 접고 다음 가지로 매끄럽게. 그게 신뢰를 산다.

## 6-2. 스타일 라우팅 — 청자 적응 디코딩 (운영자 발의 · 260713)

> "사주에서 나오는 군상을 애니어그램과 교차하면, 사람마다 먹히는 스타일이 있을 것."

- **원리**: 성격 유형론(애니어그램 등)의 쓰임은 유형 '내용'이 아니라 **발화 스타일 선택기** — 같은 계산 사실을 청자 군상에 맞는 포장으로. LLM 대응물 = **같은 샘플러 + 스타일 컨디셔닝(시스템 프롬프트)만 교체**. 내용(참인 토큰) 불변, 디코더 말투만 적응.
- **청자 프로파일 = 추가 검사 불요**: 엔진이 이미 계산 — 십신 그룹 분포(주체/수용/표현/실리/규범), 신강약, 생왕고(역마·도화·화개), 오행 편중이 곧 군상 축.
- **딜리버리 매트릭스(예시)**: 같은 사실("이별수")도 — 안정형(관성·인성 강)=대비·보호 프레임 / 변화형(역마·식상 강)=전환·해방 프레임 / 분석형=근거 선행("왜냐면") / 직진형(비겁 강)=hedge 최소·담백. 축별 {훅 강도, hedge 수위, 비유 계열(구조↔장면), 위로:도전 비율, 답변지 문구 톤}을 프리셋으로.
- **교차 확인 정곡**: 사용자가 타 체계 유형(애니어그램·MBTI 등)을 **직접 제공**하면 — "네 사주의 X와 네가 준 Y가 같은 방향을 가리킨다"는 독립 체계 교차 관찰 = 강력한 정곡. 단 '관찰'로만, 융합 주장 금지.
- **가드 2**: ① 사주→유형론 1:1 대응을 사실처럼 노출 금지(검증된 대응 없음 — 내부 휴리스틱 전용) ② 스타일 적응은 포장만 — 계산 사실을 숨기거나 뒤집지 못한다(§5 우선).
