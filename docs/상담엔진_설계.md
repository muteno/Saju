# 상담엔진 설계 — 사주 대화형 미연시 「정곡 디코더」

> **위상: 설계 정본(이데이션 수렴본).** 구현 아님 — 이 문서는 "만세력 시드 → 트리 디코드 = LLM 토큰 선택" 통찰을 실제 빌드 가능한 아키텍처로 못박는다.
> 수렴 근거 = 분신 4기 병렬 감사(엔진 실측 / 재미 메커닉 / 예타 접목 / 정직 가드레일). 각 주장은 `파일:줄` 또는 엔진 반환 필드 실측 기반.
> 데이터(해석 소스)는 이 위에 **살만 붙인다** — 아키텍처가 먼저다.

---

## 0. 한 줄 논지

**사주 상담 = 아무 정보 없는 상태에서 만세력(팔자 8글자)만 읽고 시작 → 특징을 하나씩 골라 파고드는 트리 탐색.** 이건 LLM이 토큰을 하나씩 고르는 디코딩과 **구조가 동형**이다. 우리는 그 디코딩을 *재밌게 표현*만 바꾼다 = 대화형 미연시(VN).

- **디코더 = 엔진**: `buildSaju()`가 팔자·오행·십신·신강약·지지관계·격국·대운을 계산(`manse/js/core/002-saju-engine.core.js:134`). 이게 "컨텍스트 + 로짓".
- **토큰 = 정곡(正鵠)**: 계산으로 참인 특징만 후보. 없는 건 언급 자체 금지(환각 차단).
- **샘플링 = 정곡 랭킹**: 희소성×체감성×안전성으로 매 수 1발 선택.
- **사용자 피드백 = 리랭킹**: "맞아/아니야"가 남은 후보 가중치를 갈아친다(빔서치 + human-in-the-loop). 이게 콜드리딩과 실계산 상담을 가르는 선이자 게임 장치.

## 1. 핵심 아키텍처 — 정곡 디코더 (제약 디코딩)

### 1) 3층 인식론 = 엔진 필드는 이미 두 층으로 갈라져 있다 (분신4 실측)

엔진 반환 필드를 **출처의 확신도**로 3분류한다. 이게 전 시스템을 관통하는 단일 규칙.

| 층 | 뜻 | 엔진 필드 | 성질 |
|---|---|---|---|
| **DET** (계산·확정) | 생년월일이면 틀릴 수 없는 달력 산술 | `pillars`(팔자·일간)·`counts`(오행 개수)·`sipsinCounts`·`relations`(합충)·`per.sinsal`(살 **이름**)·`unseong`·`gongmang`·`daeun/seun` 간지·괴강 여부 | `runSelfTest()` 앵커로 자가검증됨 |
| **HEU** (해석·통설) | 유파 합의 없는 휴리스틱 | `strength`(신강약)·`yongsin`(억부/조후)·`gyeokguk`·`patterns`·살의 **의미** | 엔진이 스스로 `"정본 아님·참고용"` 라벨(`005`·`009` SOURCE_NOTE) |
| **EVT** (사건·미상) | 엔진이 계산 못 하는 실제 생애 사건 | (없음) | 내담자만 앎 → **질문으로만** |

**한 필드도 쪼갠다**: 살의 *이름/존재* = DET("도화가 있다"=단정), 그 *함의* = HEU("도화라서 사람을 끈다"=추정). 이 분리가 "글자→낙인"을 기계로 막는 지점.

### 2) 토큰 어휘 = 정곡 후보 (분신1·2 실측 — 상세표 §10)

엔진 필드를 상담에서 쏠 수 있는 이산 토큰으로. 각 토큰 = {발동조건(엔진 필드식) · 층(DET/HEU) · 희소성 · 체감성}.
- 예: `counts.금===0` → 「식상 결핍」 토큰(DET·흔함·체감높음). `patterns`에 간여지동 → 「간여지동」(HEU·드묾·체감매우높음).

### 3) 정곡 랭킹 = 샘플링 (탄약 선별)

매 수마다 **참인 후보** 중 임팩트 최상위 1발:
```
impact = 희소성×0.35 + 체감성×0.40 + 안전성×0.25
```
- **가장 센 정곡 = 시기 특정**: "○살 무렵 큰 변화 있었지?" — 원국 지지 × 대운/세운 지지를 지지관계표로 돌려 충/원진/삼형 완성 **연도**를 계산(§8-②). 시점 특정이 "어떻게 알았지?!"의 최대치.
- **훅 공식**: 통념 꺾기가 제일 셈 — "신강=기 센 사람? 오해야", "사주로 부의 크기는 못 봐". 기대를 한 번 부정하고 들어가면 신뢰가 선다.

### 4) 리랭킹 = "맞아/아니야" 빔서치 게이트 (분신1·4)

**확신 판별선 = 임계 근접도.** 애매하면 단정 대신 질문으로 연다.

| 판별 | 확신(단정) | 애매(질문 필요) | 피벗 필드 |
|---|---|---|---|
| 신강약 | `score`≥50 또는 ≤30 | `score` 43~47·33~37 | `002:85`·`005:9`(임계 45/35) |
| 종격↔정격 | 일반 편차 | 극신강/극신약+오행 편중 | `score`+`counts` (**엔진 판정 결핍 → §8-①**) |
| 지지관계 | 단일 충/합 명확 | 합·충 동시(`巳申`=육합+형+파) | `relations[].rel` 길이≥2 |

**실제 리랭킹 예**: `score=46`(임계 턱걸이) → 신강 정곡 "주관 세고 안 지는 편?" → 사용자 "아니야, 오히려 휘둘려" → 신약으로 flip → `yongsin.eokbu`가 설기(식상·재·관)↔부조(인성·비겁) **완전 반전** → 조언 트리 전체 교체. **이게 실제 상담 정석**(종격/정격을 이력 대조로 가림)의 코드화.

## 2. 대화 문법 — 3단 어조 (분신4)

speech-act를 층에 묶는다. 어미가 마커.

| speech-act | 허용 층 | 종결어미 | 완충 | 금지 |
|---|---|---|---|---|
| **단정** | DET만 | `~야 ~네 ~다 ~거든` | 없음 | — |
| **hedge** | HEU | `~한 편이야 ~기 쉬워 ~일 수 있어 ~로 나오더라` | **필수**(대체로/보통/~쪽) | DET 어미 금지 |
| **질문** | EVT | `~하지 않았어? ~있었지? 혹시 ~?` | tag question | 유도·우기기 금지 |

**어미 검사기(기계 게이트)**: HEU 문장이 완충 없이 DET 어미로 끝나면 검증 실패 → hedge 자동 강등. 낙인 차단을 코드로.

**대사 예 — 괴강(일주=庚辰/庚戌/壬辰/戊戌, DET 판정)**:
- 계산(단정): "네 일주가 괴강이야. 생일로 딱 떨어지니 틀릴 수가 없어."
- 해석(hedge): "괴강은 고집·주체성 센 편이라고들 봐 — 절대는 아니고 그런 경향."
- 사건(질문): "한번 마음먹으면 끝까지 밀다가 크게 부딪힌 적 있지 않았어?"

**고리 접속사 인벤토리**: "왜냐면"(근거) → "그런데 여기서 중요한 게"(전환) → "그럼 어떻게 하냐"(처방) → 착지는 항상 본질/타이밍.

**비유 사전(툴팁 상수로 박기)**: 신강약=인풋/아웃풋 · 용신=평생 방향(마법지팡이 아님) · 오행=계절 에너지 · 재다신약="부잣집의 가난한 머슴" · 대운=연극 막 전환.

## 3. 상태기계 — 고리고리(드릴다운)

**정석 5스텝 골격 위에, 계산에서 잡힌 특이점을 맨 앞에 세우는 이중 트랙** (에세이형 통변 76편 분석 결론):
```
특징 발견(특이점 훅) → 성향 → 삶의 영역(직업·연애·재물·건강) → 시기(대운) → 조언
```
세로 고리(백미) = 한 특징이 5스텝을 관통: "金이 0이야 → 그래서 식상이 비었어 → 근데 네 용신이 바로 그 빈 金이야 → 그게 들어오는 게 N세 대운".

**PROBE 노드**(HEU·확신 낮음 → 질문으로 개방):
```
PROBE { claim_field:<HEU>, probe:"<EVT 질문>",
  on_yes  → COMMIT(트리 A)·호감+·근거 펼침·용신 심화
  on_no   → SWITCH(트리 B)·우기지 않음·대체 가설 재도출
  on_unsure → NEUTRAL·양쪽 제시·다음 노드 }
```
**"아니야" 우아하게 받기 — 철칙: DET는 안 굽힌다, HEU만 양보한다.**
- ✅ "흥, 아니라고? 계산은 맞는데 네가 안 그렇다면 결이 다른 거야. 다시 볼게." (DET 유지 + HEU 양보)
- ❌ "아냐, 넌 그래." ← 우기기 = 콜드리딩/가스라이팅 = 금지

**엔딩 문법**: 길흉 판정으로 안 끝냄 → 용신 실천 처방 + 시기 + 격려 **3종 세트**. "주눅들지 말고 칼자루를 쥐어라."

## 4. 재미 6축 — 만원짜리 밀도 (분신2)

오프라인 만원 사주가 파는 6가지에 각 축을 못박음. **6축 전부 무-LLM 결정론(seed=사주 해시)으로 완주 가능.** LLM은 리액션 승격 레이어일 뿐.

| # | 축 | 무엇 | 재활용 자산 |
|---|---|---|---|
| 1 | **정곡 크리티컬 히트** | 단정 일으킨 팔자 글자가 헤더에서 붉게 맥동 → [맞아] 순간 「的中」 인장 쾅+햅틱+"정곡 N연속🎯" 콤보. 글자가 **증거로 잠김**(콜드리딩 아님 시각화) | 팔자 8글자 프로그레스·契 인장·인주색 |
| 2 | **운명패(牌) 도감** | 플래그 확정마다 카드 **발권**. 살/귀인=희소 카드(등급 젬+"인구 X%만 가진 패"). **뽑기인데 돈이 아니라 생일이 뽑는다** → 사행성·규제 회피 + 리빌 쾌감 | **12지 컷아웃 12종 + 60갑자 60종 이미 완성** = 조달비 0 |
| 3 | **명식수(命式樹)** | 팔자=씨앗, 진행하며 가지가 뻗음. [맞아]=오행색 잎 / [아니야]=반투명 유령가지("저 길로 갔으면?"=리플레이 훅). **프로젝트 핵심 논지의 문자적 시각화** | 오행색 --el-*·목(木)=나무·수묵 산 에셋 |
| 4 | **호감도·대운 루트** | 호감도↑ → 츤 벗겨지며 속마음 해금(Lv1 퉁명→Lv5 "딱히 널 위해선…"). 대운(10년)=미연시 루트 · 하루 한 고리(daily 스트릭) | 호감도 게이지·표정 5종·어투 4층 |
| 5 | **명함(命片) 공유카드** | 9:16 = 팔자+오행바(금0⚠)+키워드칩+명대사+희소배지+마스코트. **궁합 카드로 친구 태그=초대 바이럴** | 공유카드·오행바 데모에 존재·12지 마스코트 |
| 6 | **복채(卜債)함** | 기본 무료·후함 → 심화/대운/궁합만 유료를 츤으로 자연 제안("첫 손님이니 반값"). 후원=마음만 | 복채 대사·契 인장 이미 존재 |

**성역(무료 불변)**: 카드·크리티컬 연출·명식수·기본 운세·공유카드. **유료는 오직 '깊이'.**

## 5. 페르소나 레이어 — 예타 접목 (분신3)

**두뇌는 이식, 몸통은 폐기.** 예타 서버 파이프라인(R2·Actions·dispatch·웜루프·4계정 폴오버)은 Actions 콜드스타트/구독과금/다중동시성 우회 장치인데, **정적 단일사용자 사주앱엔 그 문제 자체가 없다**(`Saju/{functions,.github}` glob=0건 실측).

| 예타 자산 | 판정 | 사주 매핑 |
|---|---|---|
| 카드 예시대화(말투 제1규칙)·tune 16축 성향 다이얼 | **이식** | 도사 카드 few-shot + "츤레벨" 게이지. `008-saju-characters.js`가 이미 골격 80% |
| 결정론 데일리 시드 `sha256(persona:date)%5` | **이식(거의 그대로)** | "오늘의 도사 기분" 공짜(순수함수) |
| `[LV n]` 관계단계·친밀도 해금 | **개념만 이식** | localStorage 방문 카운터 |
| 기억 모델(R2 CAS 노트) | **폐기** | **팔자=완벽한 결정론 기억 앵커** → 생일 한 줄만 저장, 나머지 매번 재계산. "기억"의 90%가 공짜 |
| dispatch·웜루프·4계정 폴오버·R2 CAS | **폐기** | 서버 0 = 경합 0 |

**같은 엔진, 다른 목소리**: 소수 도사 로스터(츤 언니 꼬미 / 근엄 노선비 / 젊은 무당)가 **같은 계산된 팔자를 각자 house-style로** 푼다 = 세컨드 오피니언 경제. 12지 마스코트는 **손님 정체성 토큰/리액션 스티커**로(도사 아님 — 손님 띠와 충돌 방지).
> ⚠ **합석(두 도사가 내 사주로 티격태격)** = LLM을 가장 원하고 가장 흉내내기 어려운 단 하나. v1=사전작성 쌍 결정론, LLM은 후속.

## 6. 신뢰 척추 — 가드레일 상태기계 (분신4)

우선순위 4층. 위 층이 개입하면 아래 출력은 폐기·교체.
```
L0 안전 게이트   취약·절망 감지 → 위로+전문도움(자살예방109·청소년1388 하드코딩) → 사주-낙인 금지  [선필터]
L1 팩트/금칙 게이트  팩트잠금(입력에 없는 카테고리 언급=폐기) + 금칙 검출(수명·이혼·질병·투자 단정 block) + 어미 게이트  [렌더 전]
L2 3단 어조 엔진   필드 tag → 단정/hedge/질문 + 분기 스위치
L3 떳떳함 레이어   show-your-work 근거 펼침 + "참고용" 라벨 노출
```
**폴백 사다리**: 1차 위반→엄격 리마인더 재생성 / 2차→템플릿 사다리(사람 검수=위반 불가) / 절망→L0.

**떳떳함을 파는 기능(show your work)**: 모든 단정 밑 접이식 `▸ 근거 보기` — 팔자 그리드+"계산 검증됨" 배지(`runSelfTest` 앵커)+오행 막대. **HEU엔 "참고용" 라벨 + `YONGSIN_SOURCE_NOTE` 원문 그대로 노출**(앱이 제 휴리스틱 지위를 자백). 공유카드에 "이건 계산/이건 해석" 2색 구분 = "딴 앱은 다 단정하는데 얘는 나눠줌"이 스샷 훅. **DET는 진짜 천문계산이라 당당, HEU/EVT는 "재미로 보는 사주" 고지가 덮음 — 층 분리가 고지가 계산까지 싸구려로 만들지 않게 하는 포인트.**

## 7. 아키텍처 결정 — 무-LLM 폴백이 바닥 (분신3·운영자 결정 필요)

정적 Cloudflare Pages(빌드리스) 제약. 두 갈래:
- **A. 무-LLM 유지 (v1 권장)**: 템플릿 사다리만(seed=팔자 해시). **오늘 출시 가능·0원·오프라인·안전·신규 인프라 0.** 팔자 엔진+지지 카피 이미 존재. 6축 재미 전부 여기서 완주.
- **B. Pages Function 1개 신설**: `fetch('/api/saju-voice',{팔자 facts})` → Function이 Gemini/OpenAI 호출(키=Pages env, **클라 금지**) → 도사 목소리 JSON → 2.5초 타임아웃 → 템플릿 폴백. 동기(~1-3초)라 웜루프·dispatch·R2 전부 불요.
> **LLM=베스트에포트 승격, 템플릿=바닥.** 예타가 비싸게 배운 결론이고, 사주는 팔자 결정론 덕에 그 바닥이 예타보다 훨씬 튼튼(팩트잠금이 공짜로 성립 — 엔진 반환 객체가 화이트리스트).

## 8. 엔진 배선점 — 딱 3개, 재료는 이미 엔진 안에 (분신1)

원국 계산(팔자·십신·오행·신강약 점수·지지관계·격국·대운/세운)은 **완비**. VN 트리에 필요한 신규 배선:
1. **종격/전왕격 판정** (§1-4 경계 핵심) — `score`·`counts`·`sipsinCounts` 있음, 판정 로직만 신규. 난이도 中(유파 이견 → 참고용 라벨 필수).
2. **원국×대운/세운 크로스조인 = 시기 특정** (§1-3 최강 정곡) — `branchRelations(a,b)` export·대운/세운 간지·원국 4지지 **전부 있음**, 크로스조인 루프 1함수만. 난이도 낮음.
3. **특수신살 룩업 모듈** (괴강·백호·양인·귀문·천을귀인) — `dayIdx`(60갑자)·전지지 있음, 일주간지 집합+일간×지지 표 **신규 모듈 1개**. 난이도 낮음.

부수 저마찰: 공망-궁 매칭 플래그, 신강약 borderline 플래그, 십신→키워드 브리지(`006` 미연결), 궁위 의미 라우팅(`007` 미import).
