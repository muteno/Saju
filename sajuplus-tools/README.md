# 사주플러스(saju.sajuplus.net) 게시판 → 워드 파이프라인 플레이북

검증 완료된 파이프라인 (2026-07-12). 새 세션은 이 문서대로 진행하면 시행착오 없이 바로 실행 가능.

## 전체 흐름 (결정 트리)

```
1) 익명 파이썬 크롤 시도  →  python crawl_board.py boards.json out_prefix
      수집 글 수 ≈ 게시판 페이지수×10 이면 → 완료, 4)로
      수집 글 수가 크게 모자라면 → 로그인 게이트, 2)로 (로그인해도 상한 ~3페이지/30글)
2) ★ 로그인 크롤 (권장·최속) — 쿠키 파일 방식 ★
      (a) 브라우저(claude-in-chrome)에서 sajuplus 아무 페이지나 열고 실행:
          const b=new Blob([document.cookie]); const a=document.createElement('a');
          a.href=URL.createObjectURL(b); a.download='sajuplus_cookie.txt';
          document.body.appendChild(a); a.click();
          → 쿠키 파일이 Google Drive Shared 폴더에 떨어짐. document.cookie 값을 툴 반환으로
            꺼내려 하지 말 것(가드가 [BLOCKED] 처리). 반드시 파일 다운로드로.
      (b) 이 쿠키는 .sajuplus.net 전 서브도메인(hand/face/taro/dang/name/tojung...)에 로그인 적용됨.
      (c) 크롤:  SAJU_COOKIE_FILE=<쿠키경로> python crawl_board.py boards.json out_prefix
      (d) 끝나면 쿠키 파일 삭제(세션 자격증명이라 방치 금물).
   2') 구(舊)방식 — 브라우저 async 크롤 + localStorage→둘째탭 다운로드.
       browser_crawl.js 참고. 쿠키 방식이 안 될 때만.
3) 워드 생성  →  node build_from_json.js <data.json> <meta.json>
      (이 폴더에서 최초 1회: npm install docx)
4) 검증  →  python으로 docx 내 XML 파싱 체크. Word COM 열 때는 잔여 WINWORD 프로세스 먼저 kill
```

## 사이트 구조 핵심

- 게시판 목록: `http://saju.sajuplus.net/?curjong=<jong>&cstyle=4&page=N` (GET으로 페이지네이션 동작)
- 글 링크 마커: href에 `acmode=b_s` + `curjong=<jong>` + `no=<글번호>` — no= 로 중복 제거
- 본문 컨테이너: `#boardContent` (제목/날짜/댓글수/조회수 + 본문). 댓글은 그 다음 table (미포함)
- 인코딩: 메타 charset 감지 필요 (EUC-KR 가능성)
- 사이트 예의: 요청 간 0.1~0.2초 딜레이

## 확인된 함정 (시행착오 원인들)

1. **로그인 게이트**: 익명은 게시판당 앞 1~2페이지만. goList 폼 POST를 재현해도 소용없음(세션 문제가 아니라 서버가 익명에게 목록을 비움).
2. **오래된 글 링크 제거 (뚫기 불가로 확정)**: 로그인해도 게시판 뒤쪽(오래된) 글은 목록에 제목만 있고 `<a>` 링크 자체가 없음(◇/◎ 표시). 실제 브라우저에서 클릭해도 안 열림. 아래 3가지 우회로 모두 실패 확인(2026-07-12):
   - 목록 페이지네이션: 경계(예: 초급 030) 이후 페이지는 링크 0개
   - **[이전글]/[다음글] 체인**: 이전/다음글도 같은 필터된 목록에서 생성됨. 경계글(030)은 [이전글]→029만 있고 031로 가는 [다음글]이 아예 없음 → 체인 단절
   - **사이트 검색**: 링크제거 글은 검색 결과에도 안 나옴(초급 "음양의 조화" → 0건). 단 검색 자체는 정상(기초 "천간" → 10건)
   결론: 서버가 특정 조건(포인트/등급 게이트 추정) 없이는 노출 안 함. 접근제어 우회이므로 더 시도하지 말 것. `no=`(DB 글번호)는 전 게시판 공용 카운터라 추측 불가.
3. **javascript_tool 제약**: 반환값 ~10KB 잘림 + 쿠키/쿼리스트링 포함 결과는 "[BLOCKED]" 차단. 대용량 데이터는 반환으로 못 꺼냄 → 다운로드 방식 필수.
4. **PNA 차단**: 페이지 JS에서 fetch로 127.0.0.1 로컬 서버 접근 불가(사설망 차단). 로컬 수신 서버 방식은 포기할 것.
5. **광고차단 감지 팝업**: `#moa_area` 오버레이가 주기적으로 떠서 클릭을 가로챔 → JS로 `document.querySelectorAll('#moa_area').forEach(e=>e.remove())`.
6. **docx-js**: 크롤 텍스트에 XML 비허용 제어문자 존재 → xmlClean 필수(없으면 Word에서 "파일 오류"). 한글은 Malgun Gothic 4종(ascii/eastAsia/hAnsi/cs) 지정. 제목 앞 ▽△◇◎ 마커 제거.
7. **Word COM 검증**: 이전 세션의 잔여 WINWORD 프로세스가 있으면 Open 실패 → `Get-Process WINWORD | Stop-Process -Force` 후 진행.

## 산출물 형식

- 데이터: `{ "<slug>": ["글1 전문", "글2 전문", ...] }` JSON. 각 글 텍스트는 `▽제목\n날짜\n댓글:(n)\n조회:n\n...본문` 형태(#boardContent 그대로).
- 워드: 게시판당 1개 `"<게시판명> 게시판 글모음.docx"` — 표지+수록목록+글별 새페이지(제목/메타/본문/관련링크). 이미지·댓글 미포함.
