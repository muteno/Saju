# /feed v2 — 지식 주입 트레이닝: 블라인드 시험 루프

새 정보(유튜브 등)가 오면 이 스킬로 검증·주입한다(루트 CLAUDE.md 제1법칙).
영상을 **답안지**로 삼는다: 영상이 대답하는 질문을 역추출하고, 영상을 모르는 **블라인드 학생**이
KB(엔진+색인+코퍼스)만으로 같은 수준의 답을 내는지 시험한다. 격차가 있으면 원인을 유추해
정제 반영하고, **새** 블라인드 학생으로 재시험 — 통과할 때까지. "대충 아네~ 패스" 금지.

> v1의 키 스캐너 판정(check_new_video.py)은 **전처리로 강등** — 중복 제거·자막 유무 분류만.
> 근거: 2026-07-17 도화도르 오행 영상 — 스캐너는 '중복(낮음)' 판정했지만 주장 단위 역판정에서
> 단일 소스 공백(기둥 연결성 휴리스틱)과 반대 관점 자산(기구신 반론)이 나왔다. **가치는 키가 아니라 주장에 있다.**

## 상태 기계 (영상 1편 = 독립 루프)

```
0.  [코드·무판단] 자막 전량 수집: python3 dosa-app/kb-tools/fetch_transcripts.py <vid…|--list 파일> --stage <dir>
    ✱ 수집기는 판단 금지 — 가치·신규성·중복 판정 없음. 영상 내 음성 전량 긁기가 유일 임무.
      (check_new_video.py는 이 단계에서 쓰지 않는다 — 판정 출력이 하류 에이전트를 오염시킴)
    ✱ 기보유 영상은 fulltext에서 복사(재수신 생략 = 캐시이지 판단 아님). 아직 주입 아님.
    ✱ 플래그 = 수집 품질 신호일 뿐: NO_SUBS(자막 자체 없음) · INCOMPLETE(분당 자수<200, 누락 의심)
      · THROTTLED(유튜브 429/봇확인 — 자막 유무 판정 불가, 쿨다운 후 재실행. **NO_SUBS와 혼동 금지**).
1.  [추출] 자막 분석 → 이 영상이 대답하는 질문 1~5개 역추론 + 영상의 답 요약 + 축자 근거
    → kb/exams/<vid>.json 시트 생성(스키마 아래). 질문은 시청자가 실제로 물을 형태로.
    기계 보조: python3 dosa-app/kb-tools/probe_coverage.py <시트> --exclude <vid>
2.  [1차 시험 — 블라인드] 학생 에이전트(기본 opus·effort high, 사용자 배정 존중)에게 질문만 전달.
    ✱ 블라인드 보장: 프롬프트에 영상 링크·자막·정답·채널명 절대 미포함. 학생은 STATUS 부팅
      절차와 KB 조회·리포트 도구만으로 답한다. 신규 영상은 주입 전이므로 '모름'이 구조적으로 보장됨.
    ✱ 기보유 영상이면 1차 시험 = "이미 있는 지식을 오탐 없이 찾아내는가" 시험으로 의미가 바뀐다.
3.  [채점] 별도 채점 에이전트: 학생 답 vs 영상 답 → 동등 / 부분 / 불일치.
    - '동등·재현' 판정에는 비자기(非自己) 출처의 축자 근거 ≥1건 필수 (probe_coverage로 실존 확인).
    - 학생 답이 영상과 다르지만 KB 근거로 타당하면 → 오답이 아니라 **관점차이 병기 대상**으로 분기
      (절대 원칙: 이견은 뭉개지 않고 병기. 이 경우 영상 주입 시 관점차이로 들어간다).
4.  [격차 처리 — 부분·불일치 질문만]
    4-a. 질문 자체 점검: 이 질문의 정답이 영상의 답이 맞는가? 질문이 부정확했으면 수정 후 2로.
    4-b. 질문이 맞다면 왜 격차인가 — 원인 유추(여기가 중요):
         ⓐ 지식 부재  ⓑ 있는데 못 찾음(키·별칭·색인 문제)  ⓒ 관점차이(3의 분기)
    4-c. 원인별 반영 — **정제된 형태로, 다음 모델이 오탐 없이 찾을 키에 걸어서**:
         ⓐ → check_new_video.py --save 주입 → 색인 3종 재생성(아래) → 얇은 키면 증류까지
         ⓑ → scan_interactions.py(PATTERNS)·scan_impressions.py(ELEMENT_TERMS)·aliases.json 별칭을
              **코드로** 수정 후 재실행 (기계산출물 손편집 금지)
         ⓒ → 해당 유닛 관점차이에 병기 (DISTILL_GUIDE 규칙)
5.  [재시험 = 4-0] 반영 완료 후, **이전 라운드와 다른 새 블라인드 학생**으로 2~3 반복.
    학생은 여전히 질문만 받는다(정답 미노출) — 반영된 지식을 '스스로 찾아' 답해야 통과.
    4-1/4-4. 동등 판정 → 시트 PASS, 종료.
    4-2. 실패 → 시트에 기록: 왜 안 나왔나(4-b 원인 분석이 옳았나) + 전체 프로세스 결함 점검
         + 무엇을 바꿔 재시도하는가 → 4-3. 다시 4-c 반영 후 재시험.
    ✱ 3라운드 실패 시 무한루프 금지 — 회색지대 큐로 승격(사용자 판단 요청).
```

## 시험 시트 스키마 — kb/exams/<vid>.json (커밋 대상: 세션 간 이어달리기)

```json
{ "vid": "...", "title": "...", "status": "pass|open|gray",
  "questions": [ { "id": "Q1", "q": "질문(시청자 언어)", "patterns": ["프로브용 정규식", "..."],
    "video_answer": { "summary": "영상의 답 요약", "quotes": [{"text": "축자", "src": "yt:<vid>"}] },
    "rounds": [ { "n": 1, "student": "opus/high", "blind": true,
      "answer_summary": "...", "grade": "동등|부분|불일치",
      "gap_cause": "부재|미발견|관점차이|질문결함", "action": "무엇을 반영했나" } ] } ] }
```

## 배치(채널 단위) 실행 — 격차3 운영

- 목록화: `yt-dlp --js-runtimes node --flat-playlist --print "%(id)s|%(title)s" <채널URL>/videos`
  (쇼츠 /shorts·라이브 /streams 탭 별도) → 코퍼스 대조 → 신규만.
- 유튜브 스로틀 대비: 수집은 fetch_transcripts.py의 페이싱(요청 간 1.5s+영상 간 6s) 기본값 유지,
  THROTTLED 발생 시 그 파도의 수집을 멈추고 시험·정립부터 진행(수집은 쿨다운 후 재개).
- **파도(wave) 20편 단위**: 영상 1편 = 독립 루프라 팬아웃 자유. 전체 validate·verify는 조율자가
  파도 끝에 1회(동시 실행 레이스 방지). 파도마다 커밋.
- 파도 보고 = 사용자 개입을 '예외'로 압축: ①판정표(링크|판정|메운 키|신규 용어)
  ②**회색지대 질문 3~5개**(단일 출처 등재 여부·관점 충돌·folk 경계 등 취향/기준 사안)
  ③**표본 시트 2~3편** 무작위 첨부(감사용). 사용자 답은 결정 로그·판정 지침에 명문화해 재발 방지.

## 색인 3종 재생성 (주입이 1건이라도 있으면)

```bash
python3 dosa-app/kb-tools/yt_filter.py
python3 dosa-app/kb-tools/scan_interactions.py
python3 dosa-app/kb-tools/scan_impressions.py
```

## 원칙 (v1 승계 + v2 추가)

- 주입 최종 기준은 신규성 — 단 신규성은 키 적중이 아니라 **시험 격차**로 판정한다.
- PASS 요건: 채점 에이전트의 '동등' + 축자 근거. 조율자 단독 선언 금지.
- 무속·타 역학 관점은 배제하지 않고 folk_etc 등급 주입(관점차이 병기용 — yt_filter.py 자동).
- 자막 불가(멤버십·자막 없음) 영상은 보고만; 사용자가 텍스트를 주면 수동 등록.
- 개인정보(상담 실명 등) 포함 영상은 주입 전 사용자에게 알림.
- 커밋 메시지: `지식 주입: <영상 요약> — <메운 키> (시험 R<라운드> PASS)`.
