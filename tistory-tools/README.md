# 안녕, 사주명리(yavares.tistory.com) → 워드 파이프라인

검증 완료 (2026-07-12). **블로그 전 카테고리 1,225글 + 일간별 운세 페이지 60건 전량 수집·문서화 완료.**
sajuplus-tools와 달리 **로그인/게이트 없음** — 익명 파이썬 크롤만으로 전량 수집됨.

## 스크립트

```
python scrape.py            # 단일 카테고리: 목록+글 수집 → posts.json (기본: 현묘의 사주 이야기)
python build_docx.py        # posts.json → docx 1개
python scrape_all.py [그룹] # 일괄 수집+docx (그룹: theory|unse|counsel, 생략=전부)
                            #   cat_*.json 캐시가 있으면 건너뜀 → 중단돼도 재실행으로 재개
python fix_pages.py         # 월운 허브가 링크하는 /pages/ 문서 수집·병합 (아래 함정 참조)
```

새 카테고리 추가: scrape_all.py의 CATS에 (그룹, 이름, URL경로) 한 줄 추가 후 실행.

## 사이트 구조 핵심

- 목록: `/category/<인코딩된 이름>?page=N` — 페이지당 10개, 새 글이 없으면 종료
- 목록 파싱: JSON-LD(BreadcrumbList)에서 글 id/제목(순서 보존), `li.item_category`에서 id→날짜
- 글 페이지: 제목 `h1.title_post`, 날짜/카테고리 `p.info > span`, 본문 div는 **`contents_style` 클래스로 찾을 것**
  (`tt_article_useless_p_margin`은 일부 글(#1172 등)에 없음 — 이걸로 찾으면 본문 0블록이 됨)
- 본문 파싱: div 깊이 추적으로 컨테이너 잘라낸 뒤 HTMLParser로 블록화.
  figure/figcaption/script/style 제외, blockquote→인용, li→글머리표, tr→"셀1 | 셀2", br→줄분리
- robots.txt: 카테고리/글 페이지 허용 (guestbook/manage/search만 금지)

## 확인된 함정

1. **본문 div 클래스**: `contents_style`로 찾을 것 (위 참조).
2. **/pages/ 문서**: 2026년 2월부터 월운(일간별)은 일반 글이 아니라 티스토리 "페이지"
   (`/pages/<슬러그>`) 10개로 분리되고, 카테고리에는 opengraph 링크 카드만 담긴 허브 글이 올라옴.
   페이지는 **어떤 카테고리 목록에도 안 잡힘** → 허브 글의 `data-og-source-url`에서 추출해 별도 수집
   (fix_pages.py). 새로 수집할 때 본문이 1~3블록뿐인 "○○ 모음" 글이 있으면 이 케이스 의심.
3. **목록 페이지 상한**: scrape.py의 `while page <= 200` — 운세 상위 카테고리가 64페이지라
   예전 상한 60에 걸린 적 있음. 초대형 카테고리면 상한 확인.
4. **이미지 전용 글**: 본문이 이미지뿐인 글(#1190 등)은 안내 문구로 대체 수록됨.
5. **상위/하위 카테고리**: 상위 목록은 하위 글을 포함. scrape_all.py의 PARENTS 검증이
   하위 합계와 대조해 상위 직속 글 누락을 잡아줌 (현재 세 상위 모두 직속 0 확인).

## 워드 생성 방식 (docx-js 아님)

기존 `..\자평명리학 게시판 글모음.docx`를 **zip 템플릿으로 재사용** — styles/numbering/설정을
그대로 복사하고 `word/document.xml`만 문자열로 생성. 서식이 기존 글모음 파일들과 정확히 일치한다.
- 구성: 표지(제목/부제/출처/저장일/※안내) → '수록 글 목록'(글머리표) → 글별 새페이지(Heading1 + 작성일·원문URL 메타줄 + 본문)
- 글 순서: 날짜 오름차순 (연재물 읽기 순서). 월운 페이지는 합성 id(허브id×1000+k)로 허브 글 바로 뒤에 정렬
- 텍스트는 XML 이스케이프 + 제어문자 제거 필수 (`esc()` 함수)
- 검증: minidom 파싱 + Heading1 수 == 글 수 assert 내장
