# 포스텔러 만세력 (pro.forceteller.com) 구조·디자인 스크랩

> 조사일: 2026-07-13 · 대상: https://pro.forceteller.com/profile/edit 및 연결 화면
> 목적: 만세력 입력 폼 + 사주 결과 화면의 구조/디자인 참고자료

---

## 1. 서비스 개요

- **포스텔러 만세력 2.2** — 포스텔러(forceteller)의 전문가용(pro) 무료 만세력 웹앱.
- 슬로건(메타 설명): "초심자부터 전문가까지, 모두의 만세력 포스텔러".
- 비로그인으로 전 기능(만세력 계산) 사용 가능. 로그인(소셜 5종)은 **저장 기능**에만 필요.
- 모바일 우선 설계(viewport `user-scalable=no`), 데스크톱에서는 600px 컬럼 중앙 배치.

## 2. 기술 스택

| 항목 | 내용 |
|---|---|
| 프레임워크 | React SPA (`#root`), Vite 빌드 (`/resource/<빌드해시>/assets/index-*.js`) |
| UI 라이브러리 | **MUI(Material-UI) + Emotion**(CSS-in-JS, speed mode → style 태그 비어있음) |
| 폰트 | **Pretendard Variable** (jsDelivr CDN, dynamic-subset) + `-apple-system, Roboto, sans-serif` |
| 라우팅 | 클라이언트 라우팅 (드로어 메뉴 항목이 `<a>`가 아닌 `<div>` — JS push) |
| API | `api.forceteller.com` (별도 서브도메인, CORS preflight 사용) |
| 정적 리소스 | `static.forceteller.com` (이미지), `/resource/202605271121-<해시>/` (빌드 자산) |
| 분석 | Google Analytics (gtag) |

주의: emotion speed mode 때문에 "페이지 저장"으로는 스타일이 딸려오지 않는다.
본 스크랩의 `snapshot_*.html`은 **CSSOM을 직렬화해 `<style>`로 주입**한 자립형 파일이다.

## 3. 정보 구조(라우트 맵)와 플로우

```
pro.forceteller.com
├── /profile/edit      프로필 입력 폼 (조사 기점, 비로그인 접근 가능)
├── /profile/confirm   입력 확인 단계 (지역시 보정 안내)
├── /result            만세력 결과 (핵심 화면)
├── /signin            소셜 로그인 (카카오/네이버/페이스북/구글/애플)
└── (드로어 메뉴) 홈 · 저장한 만세력(로그인 게이트) · 프로필 입력 · 자주 묻는 질문 · 고객센터
```

핵심 플로우 (3단계 위저드형):
```
프로필 입력(/profile/edit)
   └─ [만세력 보러가기] (폼 완성 전 disabled)
        ↓
입력 확인(/profile/confirm) — "입력하신 프로필을 확인해주세요."
   ├─ 요약 카드 3장: 👤이름/성별 · 📅양력 생년월일시 · 📍도시
   ├─ 안내 칩: "입력하신 지역 정보에 따라 -32분을 보정합니다." (도시 경도 기반 진태양시 보정)
   └─ [만세력 보러가기] / [프로필 수정하기]
        ↓
결과(/result) — 로딩 중 코치마크: "대운 그래프를 좌우로 슬라이드 해보세요."
```

### 4.2 폼 검증/상태

- 필수값(성별·생년월일·도시)이 채워지면 주 버튼이 회색(#E8E8E8/26% 텍스트) → **옐로(#FFD36E) + 브라운 텍스트(#524C46)**로 활성화.
- 이름 최대 12자. 날짜/시간은 텍스트 마스크 입력(양력 기본, 음력/윤달 선택).
- "시간 모름" 체크 시 시간 입력 생략 가능한 흐름.

## 6. /result (만세력 결과) 화면 구성 — 섹션 순서대로

1. **프로필 헤더**: 띠 아바타 + 이름 + "병인(붉은 호랑이) ⓠ" + 생년월일 3줄
   - `양 1990/01/01 12:00 여자 서울특별시` (회색)
   - `음(평달) 1989/12/05 12:00 …` (음력 라벨 #428BFF)
   - `양 1990/01/01 11:28 … (지역시 -32분)` (보정시각, 진하게)
2. **사주 원국 표**: 열=생시/생일/생월/생년, 행=천간(한글+한자 대형, 오행색, +목/-화 첨자)→십성→지지→십성→지장간→12운성→12신살
3. **사주 풀이 자세히 보기** (다크브라운 접이식 바): 칩 필터 — 궁성/천간합/지지육합/지지삼합/지지방합/천간충/지지충/공망/형/파/해/원진 (선택 칩만 다크 배경)
4. **신살과 길성 ⓠ**: 보유 신살 나열 + 주(柱)별 매트릭스 표(천간/길성/지지/길성, 해당 없음 ×)
5. **오행과 십성 분석**: 토글 2개("합에 따른 오행 변화 적용", "조후와 궁성 보정값 적용") + **도넛 차트 2개**(중심 텍스트 화火/비견比肩) + 표 2개
   - 오행 표: `목 25.0% 발달 / 화 50.0% 과다 / 토 12.5% 적정 / 금 0.0% 부족 / 수 12.5% 적정` — 판정어휘: 부족/적정/발달/과다
   - 십성 표: 정인~편관 10항목 %
6. **나의 오행: 병화**: 오각형 상생상극 다이어그램 — 노드 5개(원형, 오행색 반원 채움, % 표기), 화살표 파랑=생(生)/빨강=극(剋)
7. **신강/신약지수 ⓠ**: 득령❌/득지🔵/득시🔵/득세🔵 배지 + "샘플님은 **신강**한 사주입니다. 14.2%의 사람이 여기에 해당합니다." + 인구 분포 라인차트(극약~극왕 9구간, '나' 마커)
8. **용신 ⓠ**: `수(억부용신)` 카드
9. **대운수 : 2(병자) ⓠ** + "좌우로 슬라이드해서 더 볼 수 있어요."
   - **대운**(나이 2~92) / **연운**(연도) / **월운**(월) 3줄의 가로 슬라이드 타일 레일
   - 타일 구조: 상단 라벨(나이·연도·월 + 십성) → 천간 타일 → 지지 타일 → 하단(십성·12운성). 타일=한글 대자+한자 소자, 오행 배경색, 현재/선택 항목은 진보더
10. **일진 달력**: 월 네비(2026년 7월, "병오년 갑오~을미월"), 요일 헤더(일=빨강, 토=파랑), 셀=날짜+일진(한글+한자)+음력일, 절기 표기(소서 10:55), 오늘 테두리
11. **액션 3종**(원형 아이콘 버튼): 저장하기(옐로) / 프로필 재입력 / 불러오기
12. **크로스셀 링크 카드**: 포스텔러 상담소 / [NEW] 2026 신년운세 / 살풀이 운세 / 사주 총운 풀이 (→ 화살표)
13. **공유** 원형 버튼

## 7. API 표면 (관찰된 것)

| 엔드포인트 | 메서드 | 용도 |
|---|---|---|
| `https://api.forceteller.com/api/pro/cities?q=<검색어>` | GET | 도시 검색(시군구 단위, Enter 트리거) |
| `https://api.forceteller.com/api/pro/profile/saju/daeun?year=<연도>` | POST | 대운/사주 계산 (결과 화면 진입 시) |

- 프로필 상태는 클라이언트 보관(비로그인) — 결과 화면은 상태 없으면 접근 불가(입력으로 리다이렉트). `/result`로 직접 새로고침하면 `/profile/edit`로 튕김.
- 저장 기능만 로그인 요구: `/signin`에서 카카오(#FEE500 스타일 옐로)/네이버(그린)/페이스북(블루)/구글(화이트)/애플(블랙) 브랜드 버튼 스택 + "소셜 로그인 시 기본 정보만 사용" 고지.

### 7.1 인증/세션 (localStorage)

비로그인 상태에서도 `localStorage`에 토큰이 발급되어 있음:

| 키 | 용도 |
|---|---|
| `__anon_id` | 익명 사용자 식별자 |
| `access_token` | API 호출용 액세스 토큰 (익명도 발급) |
| `refresh_token` | 토큰 갱신용 |

→ 즉 **익명 세션 기반**. 비로그인 사용자도 `api.forceteller.com` 호출 시 이 토큰을 사용. 소셜 로그인은 이 익명 세션을 실계정에 승격시키는 구조로 추정.

### 7.2 진태양시(지역시) 보정 — 도시 경도 기반

확인 화면/결과 화면의 "-N분 보정" 값은 **도시마다 다름**(경도에 따른 진태양시 계산):

| 도시 | 보정값 |
|---|---|
| 서울특별시 | -32분 |
| 전라남도 순천시 | -31분 |

→ 결과 헤더에 보정 전(입력)·보정 후(실제 계산) 시각을 함께 표기. 예(황세웅 케이스):
- `양 1909/11/30 08:00 … 순천시` (입력값)
- `양 1909/11/30 07:59 … 순천시 (지역시 -31분)` (보정 후 = 실제 사주 산출 기준)
- `음(평달) 1909/10/18 08:00 …` (음력 환산)

## 9. 크롤/자동화 시 함정 메모

- SPA라 `curl`로는 빈 셸만 나옴. 렌더 후 DOM 필요 시 브라우저 경유 + **CSSOM 직렬화** 필수(emotion speed mode).
- 도시 인풋은 readOnly — 값 주입이 아니라 다이얼로그 열고 내부 input에 native setter + `input` 이벤트 + **Enter**로 검색해야 함. React 제어 컴포넌트라 단순 `.value=`는 무효.
- 이름/날짜/시간 인풋은 native setter + input 이벤트로 주입 가능.
- 결과 화면 진입 직후 광고 모달(상담소 프로모)이 클릭을 가로챔 — "닫기" 클릭 필요.
- 페이지 이동 간 렌더러가 잠깐 얼어 CDP 스크린샷 타임아웃이 간헐 발생 — 재시도하면 됨.

## 10. 참고자료로서의 시사점 (사주 사이트/문서 디자인에 차용할 것)

5. 3단계 위저드(입력→확인→결과)와 진태양시(-32분) 보정 고지 — 생시 정확성 UX의 모범.
