01 — Overview

모두의 만세력, 전문가 옵션까지

비로그인으로 만세력 계산 전 기능을 쓸 수 있고, 로그인은 “저장” 기능에만 필요하다. 모바일 우선 설계라 데스크톱에서는 600px 컬럼을 화면 가운데에 그대로 세운다. 슬로건은 “초심자부터 전문가까지, 모두의 만세력.”

프레임워크 React SPA · Vite UI MUI + Emotion (CSS-in-JS) 폰트 Pretendard Variable API api.forceteller.com 정적 static.forceteller.com 분석 Google Analytics
주의

Emotion이 speed mode라 <style data-emotion> 태그가 비어 있다 — 페이지를 그냥 저장하면 스타일이 통째로 유실된다. 이 참고자료의 스냅샷 HTML은 CSSOM을 직렬화해 <style>로 주입한 자립형이라 브라우저로 열면 그대로 재현된다.

02 — Screen Flow

3단계 위저드

입력 → 확인 → 결과. 상태는 클라이언트에 들고 있어서, /result를 직접 새로고침하면 입력 화면으로 튕긴다.

1 /profile/edit

프로필 입력

이름·성별·생년월일시·도시. 필수값이 차면 주 버튼이 회색→옐로로 활성화.

2 /profile/confirm

입력 확인

요약 카드 3장 + “지역 정보에 따라 −N분 보정” 안내. 진태양시 보정을 미리 고지.

→
3 /result

만세력 결과

원국표·오행·대운·일진 달력까지. 진입 직후 코치마크 “대운 그래프를 좌우로 슬라이드.”

보정

“−N분”은 도시 경도에 따라 값이 다르다(진태양시). 서울 −32분, 순천 −31분. 결과 헤더엔 입력 시각과 보정 후 시각을 함께 찍는다 — 예: 08:00 입력 → 07:59 (지역시 −31분)가 실제 산출 기준.

07 — Result Screen

/result 구성 — 위에서 아래로

스크롤 순서 그대로. 대부분 클라이언트 계산이고, 대운만 API를 친다.

  1. 프로필 헤더 — 띠 아바타 + 이름 + “병인(붉은 호랑이)” + 양/음/보정 시각 3줄
  2. 사주 원국표 — 열=생시/생일/생월/생년, 행=천간·십성·지지·십성·지장간·12운성·12신살. 한글 대자 + 한자 첨자 + 오행 부호(+목/−화)
  3. 사주 풀이 자세히 보기 — 칩 필터: 궁성·천간합·지지육합·삼합·방합·천간충·지지충·공망·형·파·해·원진
  4. 신살과 길성 — 보유 신살 나열 + 주(柱)별 매트릭스 표 (없으면 ×)
  5. 오행·십성 분석 — 토글 2개 + 도넛 차트 2개 + 표. 판정어휘 부족/적정/발달/과다
  6. 나의 오행 — 오각형 상생상극 다이어그램, 파랑=생(生)·빨강=극(剋)
  7. 신강/신약지수 — 득령·득지·득시·득세 배지 + 인구 분포 라인차트(극약~극왕 9구간, ‘나’ 마커)
  8. 용신 — 예: 수(억부용신)
  9. 대운수 + 대운/연운/월운 — 가로 슬라이드 타일 레일 3줄 (API 호출 지점)
  10. 일진 달력 — 월 네비 + 간지·음력일·절기 표기, 오늘 테두리
  11. 액션 3종 — 저장하기 · 프로필 재입력 · 불러오기
  12. 크로스셀 카드 + 공유 — 상담소 · 신년운세 · 살풀이 · 총운 풀이

09 — Interaction Patterns

차용할 만한 장치

도시 readOnly 검색 인풋

인풋 자체는 readOnly. 클릭하면 전폭 시트형 다이얼로그가 열리고, 내부 실검색 input에서 Enter 시 cities?q= 호출 → 결과 선택 → 원 필드에 채움.

미입력 헬퍼: “태어난 도시를 모르시면 ‘서울특별시’ 혹은 태어난 국가의 수도를.”

ⓠ 12간지 시간표

다크브라운 다이얼로그(radius 12px) + 시간표 이미지 + 팁: “자시라면 01:20, 축시라면 3:20으로 입력 권장.”

ⓠ 야자시/조자시

텍스트 팝업. 자시(23:30~)를 둘로 나눠 12시 날짜 변경을 적용하는 현대식 해석 — 23:30~00:00 출생 시 시간은 그대로, 일자만 전날로.

☰ 우측 드로어 + 광고 모달

드로어 300px, 하단 [앱 다운로드] 다크 필 버튼. 결과 진입 직후 센터 광고 모달이 뜨며 “오늘 보지 않기 | 닫기” — 자동화 시 클릭을 가로챈다.

10 — Automation Traps

크롤·자동화 시 함정

  • SPA라 curl은 빈 셸만 반환. 렌더 후 DOM이 필요하면 브라우저 경유 + CSSOM 직렬화 필수(Emotion speed mode).
  • 도시 인풋은 readOnly — 값 주입이 아니라 다이얼로그를 열고 내부 input에 native setter + input 이벤트 + Enter. React 제어 컴포넌트라 단순 .value=는 무효.
  • 이름·날짜·시간은 native setter + input 이벤트로 주입 가능.
  • 결과 진입 직후 광고 모달이 클릭을 가로챈다 — “닫기” 필요.
  • 페이지 전환 간 렌더러가 잠깐 얼어 CDP 스크린샷/평가가 간헐 타임아웃 — 재시도로 해결.
